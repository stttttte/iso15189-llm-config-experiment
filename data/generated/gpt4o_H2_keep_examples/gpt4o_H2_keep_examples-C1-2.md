# 内部质量控制程序文件审查报告

## 一、审查基本信息

- **受审文件名称**：内部质量控制程序文件  
- **程序编号**：ANA-003  
- **版本号**：1.5  
- **生效日期**：2024年12月1日  
- **审查依据**：  
  1. **ISO 15189:2022**  
  2. **CNAS-CL02:2023《医学实验室 质量和能力认可准则》**  
  3. 医学实验室内部质量控制通行要求及 Westgard 规则应用实践  
  4. 文件化管理的一般规范要求  

- **审查角色**：CNAS 主任评审员视角  
- **审查方式**：文件审查  

---

## 二、总体审查结论

该《内部质量控制程序文件》已初步覆盖内部质量控制的部分核心内容，包括质控频率、L-J 图、Westgard 规则、异常处理、室间质评和记录要求等，说明实验室已建立基本的质量控制框架。  
但从 **ISO 15189:2022 / CNAS-CL02:2023 符合性、文件规范性、逻辑一致性及可操作性** 角度看，文件仍存在较多缺陷，**目前不宜直接作为受控正式版发布使用**。

主要问题集中在以下四类：

1. **文件结构与版本控制不规范**  
2. **标准条款识别与覆盖不准确**  
3. **术语使用不符合 ISO 15189:2022 新版要求**  
4. **程序内容可操作性不足，部分要求存在逻辑冲突**

综合判断：  
**本文件审查结论为：需重大修改后重审。**

---

## 三、问题分级原则说明

本报告采用如下分级方式：

- **A级（严重）**：导致标准条款明显不符合，或程序无法有效实施，或可能直接影响检验结果质量与患者安全。  
- **B级（重要）**：虽不构成系统性失效，但影响程序完整性、可操作性或标准符合性。  
- **C级（一般）**：文件格式、术语、表述或规范性问题，不立即影响体系运行，但应纠正。

## 3. 标准引用不准确，条款识别存在偏差

- **问题编号**：NC-03  
- **等级**：**A级（严重）**  
- **位置**：第1条“目的”、全文条款覆盖  
- **发现描述**：  
  文件在目的中声明“符合 ISO 15189:2022 第 7.3.1 和 7.3.2 条要求”，但内部质量控制程序的核心依据不应仅限于此种笼统表述。  
  同时，审查意见中提出“缺少 7.3.4 测量不确定度覆盖”，这一判断本身需要进一步校正：**测量不确定度并非内部质量控制程序必须完整承载的内容**，通常应由单独的“测量不确定度评估程序”或方法学文件管理。  
  现文件的主要问题不是“必须把 7.3.4 写进本程序”，而是：**文件未清晰界定本程序与方法验证、测量不确定度、能力验证、结果有效性监控之间的接口关系**。  

- **涉及标准/要求**：  
  - ISO 15189:2022 第 7.3 条  
  - ISO 15189:2022 第 8.3 条  
  - CNAS-CL02:2023 对应条款  

- **影响评估**：  
  若标准条款映射错误，会导致实验室误判符合性，现场评审时容易出现“写了但不对口”或“该程序承诺了不该承诺的内容”。  

- **整改建议**：  
  1. 重新梳理本程序适用条款，建议明确至少关联：  
     - 检验结果有效性监控相关要求  
     - 内部质量控制实施要求  
     - 能力验证/室间质评接口要求  
  2. 将“测量不确定度”改为**关联程序引用**，例如：  
     - “测量不确定度的评估与定期复核按《测量不确定度评估程序》执行。”  
  3. 在“相关文件”中建立与以下文件的接口：  
     - 方法确认/验证程序  
     - 测量不确定度评估程序  
     - 能力验证与室间质评程序  
     - 不符合工作管理程序  

> 评审员提示：  
> 你方“问题4”的原判断方向有一定道理，但不宜简单认定为“7.3.4遗漏即本文件不符合”。更准确的表述应为：**相关接口文件和职责边界未说明，体系衔接不完整。**

## 5. 术语使用不符合 ISO 15189:2022 规范

- **问题编号**：NC-05  
- **等级**：**B级（重要）**  
- **位置**：第6.2条等  
- **发现描述**：  
  文件使用“分析过程”“标本”等旧术语，而 ISO 15189:2022 及当前 CNAS 文件体系更推荐使用“检验过程”“样本”等规范术语。  

- **涉及标准/要求**：  
  - ISO 15189:2022 术语体系  
  - CNAS-CL02:2023 对应术语要求  

- **影响评估**：

- **整改建议**：

  - “分析前/分析中/分析后”改为“检验前/检验过程/检验后”  
  - “标本”改为“样本”  
  - 如涉及“正常值范围”，应改为“生物参考区间”  
  - 如涉及“不确定度”，应规范为“测量不确定度”

## 6. 内部质控适用范围界定不完整

- **问题编号**：NC-06  
- **等级**：**A级（严重）**  
- **位置**：第2条 范围  
- **发现描述**：  
  文件明确写明“仅适用于所有定量分析检验项目”，并排除了“定性检验项目和半定量项目”。该表述存在明显风险。  
  医学实验室并非只有定量项目需要质量控制，定性和半定量项目同样应建立适宜的内部质量控制措施，例如阳性/阴性质控、临界水平质控、批内阳性质控、过程控制等。  

- **涉及标准/要求**：  
  - ISO 15189:2022 关于检验结果有效性监控的要求  
  - CNAS-CL02:2023 对应条款  

- **影响评估**：  
  该问题直接导致部分检测项目未纳入质量控制体系，属于实质性缺陷。  

- **整改建议**：  
  应将适用范围修订为：  
  - 本程序适用于实验室所有检测项目的内部质量控制；  
  - 对不同类型项目分别规定控制方式：  
    - 定量项目：L-J 图、Westgard 规则等  
    - 定性项目：阳性/阴性质控、批次质控、一致性质控  
    - 半定量项目：分级符合性监控、临界水平监控  
  若定性和半定量项目另设专门文件，应在本程序中明确引用，不得直接排除。

## 7. 质控频率要求不明确且部分设置可能不足

- **问题编号**：NC-07  
- **等级**：**A级（严重）**  
- **位置**：第3.4.2条  
- **发现描述**：  
  文件规定“每个检验项目开始分析前，应至少进行1个水平的质控；生化和免疫项目应进行高、中、低三个浓度水平质控；8小时班次每4小时1次”。  
  该规定存在以下问题：  
  1. “所有定量项目至少1个水平”不一定充分；  
  2. 未基于检测项目风险、仪器稳定性、批量大小、校准稳定性和厂家说明书设定频率；  
  3. 未规定随机检测、连续上机、大批量检测、24小时运行设备的质控策略；  
  4. 未明确失控后恢复检测前的放行判据。  

- **涉及标准/要求**：  
  - ISO 15189:2022 关于监控检验结果有效性的要求  
  - CNAS-CL02:2023 对应要求  

- **影响评估**：  
  质控频率不合理会导致失控不能被及时发现，直接影响结果放行安全性。  

- **整改建议**：  
  应基于项目风险制定分层要求，至少明确：  
  - 每台设备、每个检测系统、每个批次/班次所需质控水平数  
  - 新批号试剂、校准后、维护后、关键部件更换后必须加做质控  
  - 长时间连续运行项目的加测间隔  
  - 失控后恢复条件：至少连续满足既定放行规则后方可恢复患者样本检测/结果审核

## 9. 异常质控处理程序不完整，未明确结果追溯与风险控制

- **问题编号**：NC-09  
- **等级**：**A级（严重）**  
- **位置**：第6条  
- **发现描述**：  
  文件在质控不合格时要求“暂停分析、重新检验、原因调查、纠正措施、重新验证”，基本方向正确，但严重缺少以下关键要素：  
  - 受影响时间段的界定  
  - 已检测未报告样本如何处理  
  - 已报告结果是否需追溯、复核、召回、更正  
  - 临床沟通机制  
  - 事件记录与不符合联动  
  - 责任岗位和时限要求  

- **涉及标准/要求**：  
  - ISO 15189:2022 关于不符合工作、结果有效性、纠正措施的相关要求  
  - CNAS-CL02:2023 对应要求  

- **影响评估**：  
  该问题直接影响患者结果风险控制，是关键缺陷。  

- **整改建议**：  
  补充失控处置闭环，至少包括：  
  1. 立即停止该检测系统结果放行；  
  2. 界定受影响的最后一个在控点至失控发现点之间的样本；  
  3. 暂停审核/报告；  
  4. 对已报告结果进行风险评估；  
  5. 必要时启动结果更正和临床通知；  
  6. 记录于不符合/失控处理表；  
  7. 纠正措施完成并验证有效后方可恢复。

## 10. 室间质评/能力验证条款表述存在逻辑冲突

- **问题编号**：NC-10  
- **等级**：**B级（重要）**  
- **位置**：第7条  
- **发现描述**：  
  文件规定“室间质评每月进行一次”，同时又写“参加经认可的能力验证计划，按 ANA-004 执行”。两者逻辑未协调。  
  一般而言：  
  - **外部能力验证/室间质评**的频率由提供者计划决定，不是实验室自行规定“每月”；  
  - 实验室内部可建立**替代比对/实验室间比对/留样比对/分样比对**等补充监控机制，但不应混同为外部 PT/EQA。  

- **涉及标准/要求**：  
  - ISO 15189:2022 关于能力验证和结果有效性监控的要求  
  - CNAS-CL02:2023 对应要求  

- **影响评估**：  
  容易造成实施偏差，也会在评审中被质疑对 EQA/PT 概念理解不清。  

- **整改建议**：  
  建议改为分层表述：  
  - **外部能力验证/室间质评**：按认可的提供者计划参加；  
  - **当无合适 PT/EQA 时**：采用实验室间比对、留样复测、替代评价等方式；  
  - 如实验室内部开展月度比对，应明确其名称为“内部比对/实验室间比对”，不得表述为“每月室间质评”。

## 七、综合结论

该程序文件**不建议按当前版本直接纳入认可体系正式运行**。  
从认可评审风险角度看，最需要优先整改的是以下 **5项A级问题**：

1. **标准引用与程序边界不清（NC-03）**

3. **质控频率及放行逻辑不充分（NC-07）**  
4. **质控失控后的结果风险控制不完整（NC-09）**

建议实验室按以下顺序整改：

### 第一优先级：先补“能不能执行”
- 表单附录
- 失控处理闭环
- 质控频率和放行规则
- 定性/半定量项目纳入范围

### 第二优先级：再补“是否符合标准”
- 标准引用与接口文件
- 能力验证/PT与内部比对的关系
- 质量指标定义

### 第三优先级：最后统一“文件规范”
- 术语修订
- 编号连续
- 修订记录标准化
- 缩略语定义
