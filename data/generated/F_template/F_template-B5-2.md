# 不符合项整改计划及验证方案（二）

## 设备维护记录不完整 —— 缺少维护后验证结果和复核签名

---

## 二、不符合事实描述

对生化分析仪 AU5800 的 2025 年度维护记录进行逐份检查，发现以下问题：

| 统计项 | 数据 |
|--------|------|
| 2025 年设备维护总次数 | 35 次 |
| 缺少维护后验证结果的记录数 | 8 次（占 22.9%） |
| 缺少技术主管/维护负责人复核签名的记录数 | 3 次（占 8.6%） |

**具体问题**：
1. 维护记录表中仅有维护日期和维护人员初次签名，缺少维护完成后设备是否恢复正常运行的验证记录（如：故障是否排除、校准是否通过、质控数据是否在控）。
2. 部分维护记录无技术主管或维护负责人的复核签名，无法确认维护工作的有效性已被审查。
3. 现行《设备维护与定期检查程序》（EQP-004）的维护记录表单中未设置"维护后验证"栏位。

---

## 三、根本原因分析

### 3.1 分析方法

采用鱼骨图（人、机、料、法、环）和"5-Why"法进行根因分析。

### 3.2 鱼骨图分析

```
                         设备维护记录不完整
                              │
    ┌──────────┬──────────┼──────────┬──────────┐
    │          │          │          │          │
   人员       方法       表单       监督       外部
    │          │          │          │          │
  维护人员   EQP-004    记录表单   质量部门   外包维护
  对记录要   未明确要   无"验证   未定期审   人员对我
  求理解不   求填写维   结果"栏   查维护记   方记录要
  够充分     护后验证              录完整性   求不了解
             结果
```

### 3.3 根因总结

1. **程序缺陷**：EQP-004 规定了维护内容和维护人员签名要求，但未明确要求记录维护后的验证结果，也未规定验证方法和合格标准。
2. **表单设计缺陷**：维护记录表单中缺少"维护后验证"专用栏位，导致即使维护人员进行了验证也无处记录。
3. **复核机制弱化**：技术主管的复核签名流于形式，未建立明确的复核内容和时限要求。
4. **外包维护管控不足**：对外包维护单位（如设备厂商工程师）的记录完整性要求未明确传达。

## 四、纠正措施计划

### 措施 1：修订《设备维护与定期检查程序》（EQP-004）

| 项目 | 内容 |
|------|------|
| **措施编号** | CA-002-1 |
| **责任人** | 设备管理部门主任 + 质量管理部门 |
| **完成期限** | 2026 年 2 月 28 日 |

**修订要点**：

**(1) 新增"维护后验证"要求**

在程序第 6 章"维护实施"部分新增以下条款：

> 每次设备维护完成后，维护人员应对设备进行功能验证，以确认设备已恢复正常运行状态。验证方法和合格标准如下：

| 维护类型 | 验证方法 | 合格标准 |
|---------|---------|---------|
| 故障维修 | 执行设备自检程序，运行诊断测试 | 故障代码已清除，自检全部通过 |
| 预防性维护 | 运行质控品，检测已知浓度样本 | 质控结果在控（在 2SD 范围内） |
| 校准维护 | 执行校准程序，验证校准曲线 | 校准斜率和截距在允许范围内 |
| 软件升级 | 运行系统功能测试 | 各功能模块运行正常，数据传输无误 |
| 日常保养 | 检查运行参数和清洁状态 | 参数正常，无异常报警 |

**(2) 明确复核签名要求**

> 维护记录填写完成后，应在 2 个工作日内由技术主管或设备管理负责人进行复核。复核内容包括：维护内容是否完整、验证方法是否适当、验证结果是否合格、记录填写是否准确。

**(3) 明确外包维护记录要求**

> 外包维护完成后，我方设备管理人员应当场核查外包维护报告的完整性，要求外包方在维护报告中包含：维护项目清单、维护后验证结果、工程师签名。我方设备管理人员签署验收确认。

### 措施 2：重新设计维护记录表单

| 项目 | 内容 |
|------|------|
| **措施编号** | CA-002-2 |
| **责任人** | 设备管理部门 |
| **完成期限** | 2026 年 2 月 28 日 |

**新表单结构**：

表单编号：FRM-EQP-MR-V2（替代原版本）

新增以下栏位：

| 栏位名称 | 填写内容 | 填写人 |
|---------|---------|--------|
| 维护后验证方法 | 具体描述所采用的验证方式（如：运行 QC、自检、校准验证等） | 维护人员 |
| 维护后验证结果 | 验证的具体数据或结论（如：QC 结果在控、自检通过等） | 维护人员 |
| 验证结论 | 合格 / 不合格（不合格需说明后续处理） | 维护人员 |
| 维护人员签名/日期 | 确认维护和验证工作完成 | 维护人员 |
| 复核意见 | 对维护内容和验证结果的审查意见 | 技术主管 |
| 复核人签名/日期 | 确认复核完成 | 技术主管 |

### 措施 3：补充 2025 年度不完整的维护记录

| 项目 | 内容 |
|------|------|
| **措施编号** | CA-002-3 |
| **责任人** | 设备管理部门 + 技术主管 |
| **完成期限** | 2026 年 3 月 15 日 |

**具体步骤**：

1. 逐一审查 2025 年度 35 份维护记录，识别全部不完整记录。
2. 对 8 条缺少验证结果的记录：
   - 回溯维护当天的设备运行日志、QC 记录、自检报告等原始数据。
   - 补充填写维护后验证情况说明（在原记录上附加补充页）。
   - 由当时的维护人员或当值技术人员签名确认，注明补记日期。
3. 对 3 条缺少复核签名的记录：
   - 由技术主管补签复核意见，注明补签日期和原因说明。
4. 所有补充记录统一编制《2025 年度维护记录补充清单》，汇总存档。

**注意**：补充记录时应真实反映当时情况，不得编造数据；若原始验证数据已无法追溯，应如实记录"原始验证数据不可追溯，但设备后续运行正常"并附相关运行日志。

### 措施 4：建立维护记录二级审核制度

| 项目 | 内容 |
|------|------|
| **措施编号** | CA-002-4 |
| **责任人** | 设备管理部门主任 |
| **启动日期** | 2026 年 2 月 1 日起执行 |

**制度内容**：

1. **第一级（维护人员）**：维护人员完成维护后，在记录表中填写全部信息（维护内容、验证方法、验证结果、签名日期），确保无空白栏位。
2. **第二级（技术主管）**：技术主管在 2 个工作日内完成复核，检查项包括：
   - 维护内容是否覆盖计划维护项目
   - 验证方法是否合适
   - 验证结果是否在合格范围
   - 记录填写是否完整准确
3. **月度汇总**：设备管理部门每月末填写《维护记录复核进度跟踪表》，统计当月维护次数、记录完成率、复核完成率，提交质量管理部门。
4. **异常处理**：对维护后验证不合格的，应立即启动设备停用 -> 再维护 -> 再验证流程，经验证合格后方可恢复使用。

### 措施 5：加强外包维护质量控制

| 项目 | 内容 |
|------|------|
| **措施编号** | CA-002-5 |
| **责任人** | 设备管理部门 |
| **完成期限** | 2026 年 2 月 15 日 |

**具体措施**：

1. 向所有外包维护单位发送正式函件，明确我方对维护记录的完整性要求，附《维护记录模板》供外包方参考。
2. 编制《外包维护验收清单》，每次外包维护结束后由我方设备管理人员当场核查以下项目：

| 核查项 | 要求 |
|--------|------|
| 维护报告 | 有维护项目明细、维护前后对比 |
| 验证记录 | 有维护后验证方法和结果 |
| 工程师签名 | 外包方维护工程师签名和日期 |
| 配件更换记录 | 如有更换部件，需记录型号、序列号 |
| 我方验收签名 | 设备管理人员确认验收并签名 |

3. 在下次外包维护到场时，对外包维护工程师进行记录要求的现场讲解（15 分钟）。
4. 将外包维护记录完整性纳入供应商年度评价指标。

## 五、验证方案

### 5.1 验证目标

确认以下整改效果：
- 新增维护记录 100% 包含验证结果和复核签名
- 2025 年度历史缺陷记录已全部补充完善
- 维护记录二级审核制度持续有效运行
- 外包维护记录符合要求

### 5.2 验证方法与标准

| 序号 | 验证内容 | 验证方法 | 判定标准 |
|------|---------|---------|---------|
| 1 | 新维护记录完整性 | 逐份检查 2026 年 2 月以后的所有新增维护记录 | 验证结果填写率 100%，复核签名签署率 100% |
| 2 | 历史记录补充情况 | 检查 8 条缺验证结果 + 3 条缺复核签名的记录 | 全部已补充完整，有补记日期和签名 |
| 3 | 维护后验证执行质量 | 抽查新增维护记录中的验证数据 | 验证方法与维护类型匹配，数据真实可追溯 |
| 4 | 复核审查实质性 | 访谈技术主管，了解复核时是否实质审查 | 技术主管能说明复核要点，不是简单签字 |
| 5 | 外包维护记录合规性 | 检查外包维护记录及我方验收记录 | 记录完整，验收清单签字齐全 |
| 6 | 二级审核制度运行 | 检查《维护记录复核进度跟踪表》 | 连续 3 个月均按时完成，完成率 >= 95% |

## 六、预防措施

为防止同类问题再次发生，额外实施以下预防措施：

1. **维护记录完整性纳入月度质量指标**：设定"维护记录完整率"指标（目标 >= 98%），每月统计并在质量月报中通报。
2. **维护人员专项培训**：每年对设备维护相关人员（含外包方）进行一次记录规范培训，内容覆盖 ISO 15189:2022 第 6.4.4 条的要求。
3. **纳入年度内审重点**：将设备维护记录完整性作为年度内审的必查项目。
4. **信息化辅助**：评估在设备管理系统中增加维护记录电子化模块的可行性，实现维护记录的在线填写、审核和预警提醒（如超过 2 个工作日未复核则自动提醒）。
