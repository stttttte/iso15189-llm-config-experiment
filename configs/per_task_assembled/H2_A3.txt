# ISO 15189:2022 检验科文件体系生成指南

## 本套模板的用途

本套 MD 模板文件用于指导大语言模型 (LLM) 为医学实验室（检验科）生成符合 **ISO 15189:2022** 要求的质量管理体系文件。

**定位**：将"从零编写"变为"审核修改"，节省 70-80% 编写时间。

**输出物**：质量手册 + 程序文件(SOP) + 记录表单/作业指导书（三层文件结构）。

---

## 推荐使用的 LLM 模型

由于单个模板文件约 8K-12K tokens，请使用以下级别的模型：

| 推荐级别 | 模型 | 说明 |
|---------|------|------|
| 最佳 | Claude Opus / Claude Sonnet | 长上下文处理能力强，中文质量高 |
| 推荐 | GPT-4 / GPT-4o | 指令遵循度好 |
| 可用 | 其他 128K+ 上下文模型 | 需测试中文输出质量 |
| 不推荐 | GPT-3.5 / 4K 上下文模型 | 上下文不足，生成质量差 |

---

## 使用流程

```
步骤1：填写下方「检验科基本信息」
  ↓
步骤2：将本文件 + 01-质量手册-框架.md 一起发给 LLM
  ↓  LLM 生成质量手册初稿
步骤3：逐个发送 02~08 程序文件模板给 LLM
  ↓  每次附上本文件的「基本信息」部分作为上下文
  ↓  LLM 逐模块生成程序文件
步骤4：发送 09-记录表单清单.md 给 LLM
  ↓  LLM 生成配套表单
步骤5：发送 10-POCT专项.md（如有 POCT）
  ↓
步骤6：人工审核 → 签批发布 → 实施运行
```

**重要提示**：
- 每次发给 LLM 时，请把下方「基本信息」一并粘贴，确保 LLM 知道你的实验室具体情况
- 生成结果中所有 `【】` 包裹的内容为必须替换的占位符
- 所有标注 `⚠️ 示例，请替换` 的文本为示例内容，必须替换为真实信息

## 给 LLM 的指令

```
请根据以下 7 个 SOP 模板框架和用户提供的检验科基本信息，逐一生成完整的程序文件。

要求：
1. 严格覆盖每个 SOP 模板中列出的全部 ISO 15189:2022 条款要求
2. 每个 SOP 按照统一格式编写（封面 → 目的 → 适用范围 → 职责 → 术语 → 程序内容 → 相关文件 → 记录表单 → 条款覆盖自查表）
3. 所有 ⚠️ 标注的示例段落必须替换为实验室真实内容
4. 所有【】占位符必须填写实际信息
5. 语气：正式、第一人称（"本实验室"）
6. 输出格式：Markdown
7. 每个 SOP 末尾必须包含「配套记录表单清单」和「条款覆盖自查表」
```

## 1. 目的

规范本实验室检验前过程的管理，确保从检验申请、样本采集、运输到接收和预处理的每个环节均得到有效控制，保障检验结果的准确性和可靠性。

---

## 2. 适用范围

适用于本实验室检验前过程的全部活动，包括：
- 患者和用户信息的提供
- 检验申请的管理
- 样本的采集
- 样本的运输
- 样本的接收
- 样本验收例外处理
- 检查前的处理、准备和储存

---

## 3. 职责

| 角色 | 职责 |
|------|------|
| 科室负责人 | 批准检验前过程管理相关制度 |
| 质量负责人 | 监督检验前过程的实施，处理相关不合格事件 |
| 技术负责人 | 制定和维护采样手册，确定样本接受/拒绝标准 |
| 各专业组组长 | 执行样本接收核查，处理不合格样本 |
| 采样人员 | 按照规范执行样本采集 |
| 前处理人员 | 执行样本的预处理、分发和储存 |

| 术语 | 定义 |
|------|------|

## 5. 程序内容

### 5.1 患者和用户信息 (§7.2.1)

> **ISO 15189:2022 原文要求**：实验室应向患者和用户提供有关实验室的信息，包括位置、工作时间、提供的活动和检查范围、预期结果报告时间、费用（如适用）、咨询服务、患者知情同意要求、投诉流程等。

本实验室通过以下方式向患者和用户提供信息：

**a) 实验室位置和工作时间**

⚠️ 示例，请替换：
> 本实验室位于【医院名称】门诊楼 3 层及住院部 1 层。工作时间为：
> - 门诊检验：周一至周五 07:30-17:00，周六 08:00-12:00
> - 急诊检验：24 小时值班
> - 联系电话：【填写】

**b) 活动范围和可提供的检查项目**

本实验室在门诊大厅、检验科候诊区公示检验项目目录，包括项目名称、标本类型、参考区间和报告时间。项目目录由技术负责人负责定期更新。

**c) 预期结果报告时间**

⚠️ 示例，请替换：

| 检验类别 | 常规报告时间 | 急诊报告时间 |
|---------|------------|------------|
| 临床化学（生化） | 当日 16:00 前 | 2 小时内 |
| 血常规 | 当日 12:00 前 | 30 分钟内 |
| 凝血功能 | 当日 14:00 前 | 2 小时内 |
| 尿液常规 | 当日 12:00 前 | 1 小时内 |
| 微生物培养 | 3-5 个工作日 | 不适用 |
| 免疫学检测 | 当日 16:00 前 | 2 小时内 |

**d) 咨询服务**

本实验室提供以下咨询服务：
- 检验项目的选择建议
- 检验结果的临床解释
- 检验方法的技术咨询
- 样本采集与运输指导

咨询服务联系方式：电话 【填写】，或通过医院 OA 系统提交咨询请求。

**e) 患者知情同意要求**

对于以下检验活动，应获得患者知情同意：
- ⚠️ 示例，请替换：HIV 抗体检测、基因检测、骨髓穿刺标本采集等

**f) 投诉流程**

患者和用户可通过以下渠道提出投诉：
- 检验科接待窗口
- 医院投诉热线：【填写】
- 医院官网在线投诉平台

投诉处理流程详见《不合格工作管理程序》(JYK-SOP-07-006)。

### 5.2 检验申请管理 (§7.2.2)

> **ISO 15189:2022 原文要求**：检验申请应包含足够的信息以识别患者和授权的申请者，以及所需的临床信息。实验室应有口头请求的管理程序。

**a) 检验申请单要求**

检验申请单（电子或纸质）应至少包含以下信息：

| 序号 | 必填信息项 | 说明 |
|------|-----------|------|
| 1 | 患者唯一标识 | 住院号/门诊号/身份证号 |
| 2 | 患者姓名 | 与身份证件一致 |
| 3 | 患者性别 | - |
| 4 | 患者出生日期/年龄 | - |
| 5 | 申请医生姓名 | 授权申请者 |
| 6 | 申请科室 | - |
| 7 | 申请日期和时间 | - |
| 8 | 检验项目 | 明确列出所申请的检验项目 |
| 9 | 临床诊断/症状 | 用于结果解释 |
| 10 | 标本类型 | 血清/全血/尿液等 |
| 11 | 特殊用药信息 | 如抗凝药物使用情况 |
| 12 | 优先级别 | 常规/急诊/加急 |

**b) 口头请求管理**

- 口头请求仅限紧急情况使用（如：急诊抢救）
- 口头请求必须由授权人员接收并记录
- 口头请求须在 【24/48】 小时内补交正式检验申请单
- 口头请求记录内容包括：请求者姓名、请求时间、患者信息、检验项目、接收人姓名
- 口头请求记录于《口头申请登记表》(JYK-FM-07-001-01)

### 5.3 样本采集 (§7.2.3)

> **ISO 15189:2022 原文要求**：实验室应有样品采集的文件化程序。采样手册应包括采集前准备、患者识别、采集程序和采集后处理的指导。

#### 5.3.1 采样手册

本实验室编制《采样手册》作为样本采集活动的指导文件，采样手册至少包含以下内容：
- 采集前患者准备信息（禁食、禁药等要求）
- 各类标本的采集程序（静脉血、动脉血、尿液、粪便、体液等）
- 采集容器/试管的选择
- 标本标识和标记要求
- 采集顺序（如多管采血的顺序）
- 标本运输和储存条件

#### 5.3.2 采集前信息和准备

采集前应核实以下信息：

| 核实项目 | 具体内容 |
|---------|---------|
| 患者身份验证 | 至少使用两种标识（姓名+住院号/门诊号），与检验申请单核对 |
| 禁食状态 | 需空腹检验项目应确认患者空腹时间 ≥ 【8-12】小时 |
| 用药情况 | 确认影响检验结果的用药信息（如抗凝药、降糖药等） |
| 采集时间 | 有时间要求的项目确认采集时间（如皮质醇、药物浓度监测） |
| 患者同意 | 特殊项目获取知情同意 |

#### 5.3.3 患者知情同意 (§7.2.3)

- 需要患者知情同意的检验项目清单由技术负责人审批
- 知情同意书保存于患者病历中
- 无法获得患者本人同意时，应获取法定代理人同意

#### 5.3.4 采集活动

**身份验证：**
- 门诊患者：核对检验申请单上的姓名、门诊号，要求患者自述姓名
- 住院患者：核对腕带信息（姓名、住院号）与检验申请单一致
- 无法自述者：由陪护人员协助核对

**采样容器选择：**

### 5.4 样本运输 (§7.2.4)

> **ISO 15189:2022 原文要求**：实验室应有确保样品在规定的时间间隔和温度条件下运输的程序，以维护样品的完整性，并保护运输人员、公众和接收实验室的安全。

**a) 运输包装要求**

- 所有标本应放置于防漏、密封的运输容器中
- 运输容器应标识生物危害标志
- 玻璃容器应有防碎保护
- 运输箱内附吸收性材料

**b) 时间要求**

⚠️ 示例，请替换：

| 标本类型 | 运输时限 | 备注 |
|---------|---------|------|
| 常规血液标本 | 采集后 2 小时内送达 | - |

| 室温 (18-25°C) | 大多数常规标本 |

- 外送标本应按照相关法规进行包装和运输

## 5. 程序内容

### 5.1 方法选择原则 (§7.3.1)

> **ISO 15189:2022 原文要求**：实验室应选择经过验证或确认适合预期用途的检查程序，优先使用 IVD 医疗器械说明书中的方法、已发表的权威方法或国际/国家标准方法。

方法选择优先顺序：

| 优先级 | 方法来源 | 验证/确认要求 |
|--------|---------|-------------|
| 1（首选） | IVD 医疗器械制造商说明书方法 | 方法验证 |
| 2 | 已发表的权威期刊方法 | 方法确认 |
| 3 | 国际/国家标准方法 | 方法验证/确认 |
| 4 | 实验室自建方法（LDT） | 方法确认（全面） |

方法选择应考虑：
- 临床需求和预期用途
- 方法的分析性能是否满足临床要求
- 样本类型和量的要求
- 检测通量和周转时间
- 试剂和设备的可获得性
- 人员技能要求
- 安全和环境影响

---

### 5.2 方法验证程序 (§7.3.2)

> **ISO 15189:2022 原文要求**：实验室应验证其使用的检查程序能够达到制造商或方法发布者声称的性能规格。验证应尽可能使用适当的样品（患者样品）。

#### 5.2.1 验证方案

方法验证前应制定验证方案，内容包括：
- 验证目的
- 验证项目（性能参数）
- 验证方法和实验设计
- 样本要求
- 接受标准
- 时间计划

#### 5.2.2 验证性能参数

**a) 精密度验证**

⚠️ 示例，请替换：
> 批内精密度：使用两个浓度水平的质控品，每个浓度水平连续测定 20 次，计算 CV%。
> 批间精密度：使用两个浓度水平的质控品，连续 20 天，每天测定 1 次，计算 CV%。
> 接受标准：CV% ≤ 制造商声称的精密度指标或 ≤ 1/3 TEa。

**b) 正确度验证**

⚠️ 示例，请替换：
> 使用有证参考物质 (CRM) 或能力验证 (PT) 样品，测定 3 次取均值，计算偏倚 (Bias)。
> 或与参考方法/同类已验证方法进行比对，测定至少 20 例患者样本。
> 接受标准：Bias ≤ 制造商声称的正确度指标或 ≤ 1/2 TEa。

**c) 线性范围验证**

⚠️ 示例，请替换：
> 配制 5-7 个浓度水平的系列样品（覆盖声称的线性范围），每个浓度测定 2-3 次。
> 以预期值为横轴、实测值为纵轴作线性回归。
> 接受标准：回归方程斜率在 0.95-1.05 之间，r² ≥ 0.995。

**d) 检出限 (LoD) / 定量限 (LoQ) 验证**

⚠️ 示例，请替换：
> 检出限：测定空白样品 20 次，LoD = 空白均值 + 2SD（或按 CLSI EP17 方法）。
> 定量限：测定低浓度样品 20 次，LoQ 为 CV ≤ 20% 时的最低浓度。

**e) 参考区间验证**

详见 5.5 节。

#### 5.2.3 验证报告

验证完成后编写《方法验证报告》(JYK-FM-07-002-01)，包括：
- 验证项目和方法概述
- 实验数据和统计分析
- 与接受标准的比较
- 结论（通过/未通过）
- 技术负责人审批签字

---

### 5.3 方法确认程序 (§7.3.3)

> **ISO 15189:2022 原文要求**：实验室应确认以下方法：a) 实验室设计或开发的方法；b) 在预期范围之外使用的方法；c) 经修改的已验证方法。确认应尽可能全面和客观。

以下情形需进行方法确认：

| 情形 | 确认内容 |
|------|---------|
| 实验室自建方法 (LDT) | 全部性能参数 + 临床灵敏度/特异性 |
| 超出制造商声称范围使用 | 超出部分的性能验证 |
| 对已验证方法进行修改 | 修改部分的影响评估 + 相关性能验证 |

方法确认除包含方法验证的全部项目外，还应包括：
- 临床灵敏度和临床特异性
- 干扰试验
- 交叉反应评估
- 与现有方法（金标准）的一致性比较
- 预期使用范围的适用性评估

确认报告使用《方法确认报告》(JYK-FM-07-002-02)。

---

### 5.4 测量不确定度评定 (§7.3.4)

> **ISO 15189:2022 原文要求**：实验室应确定每个测量程序的测量不确定度，并定义其性能规范。实验室应定期审查测量不确定度。

#### 5.4.1 MU 评估方法

本实验室采用以下方法评估测量不确定度：

**自上而下 (Top-down) 方法**（推荐）：

⚠️ 示例，请替换：
> 利用室内质控数据和室间质评数据评估 MU：
> - 批内不精密度 (u_rw)：来源于长期 IQC 数据的 SD
> - 偏倚不确定度 (u_bias)：来源于 EQA/PT 数据或与参考物质的比对
> - 合成标准不确定度：u_c = √(u_rw² + u_bias²)
> - 扩展不确定度：U = k × u_c（k=2，覆盖因子，约 95% 置信水平）

#### 5.4.2 MU 与性能规范的比较

- 评估的 MU 应与临床需要的性能规范进行比较
- 当 MU 超过性能规范时，应采取改进措施
- 性能规范可参考：生物学变异数据、专家共识、法规要求

#### 5.4.3 定期审查

- MU 应至少每 【年/半年】 审查一次
- 方法、设备、试剂发生重大变更时应重新评估 MU
- MU 审查记录于《测量不确定度评估记录》(JYK-FM-07-002-03)

---

### 5.5 生物学参考区间和临床决策极限 (§7.3.5)

> **ISO 15189:2022 原文要求**：实验室应定义其报告检查结果所使用的生物学参考区间或临床决策极限，并将其文件化。

#### 5.5.1 参考区间来源

| 来源 | 适用条件 |
|------|---------|
| 制造商说明书 | 经本实验室验证适用 |
| 国家/行业标准 | 权威指南推荐 |
| 文献发表值 | 经本实验室验证适用 |
| 本实验室建立 | 无可用参考值或有特殊人群需求 |

#### 5.5.2 参考区间验证

⚠️ 示例，请替换：
> 按照 CLSI EP28-A3c 方法：
> - 选择至少 20 名健康参考个体
> - 使用本实验室方法和设备进行检测
> - 如 20 例中 ≤ 2 例超出参考区间，则验证通过
> - 如 > 2 例超出，增加样本量至 60 例或考虑建立本实验室参考区间

#### 5.5.3 定期审查

- 参考区间应至少每 【年】 审查一次
- 以下情况需重新评估参考区间：
  - 检验方法或设备变更
  - 试剂批号更换后性能出现显著变化
  - 室间质评数据提示偏倚变化
  - 临床反馈提示参考区间不适当

---

每个检验项目应编制标准作业指导书 (WI)，至少包含：

| 内容项目 | 说明 |

| 安全注意事项 | 生物安全、化学安全 |

### 5.7 方法变更管理 (§7.3.1)

> **ISO 15189:2022 原文要求**：当检查程序发生变更时，应进行影响评估并通知相关用户。

**a) 方法变更的情形**

- 检测方法更换（如方法原理改变）
- 仪器设备更换或重大维修
- 试剂品牌/系统更换
- 检测流程的重大调整

**b) 变更管理流程**

```
变更申请 → 影响评估 → 验证/确认 → 审批 → 用户通知 → 实施 → 效果跟踪
```

- 变更前进行影响评估，评估对患者结果的影响
- 按照 5.2 或 5.3 进行方法验证/确认
- 变更经技术负责人/科室负责人审批
- 通知临床科室等相关用户（包括参考区间变更通知）
- 变更记录于《方法变更管理记录》(JYK-FM-07-002-04)

## 5. 程序内容

### 5.1 IQC 材料选择 (§7.3.7.2)

> **ISO 15189:2022 原文要求**：实验室应使用质量控制材料，在临床相关的浓度水平上定期验证检查程序的持续可靠性。质量控制材料应尽可能类似于患者样品。

**a) IQC 材料要求**

| 选择标准 | 要求 |
|---------|------|
| 基质效应 | 质控品基质应尽可能接近患者样本 |
| 稳定性 | 应有足够的保质期和开瓶稳定性 |
| 浓度水平 | 应至少包含两个浓度水平（正常和异常水平） |
| 临床决策水平 | 浓度水平应覆盖临床决策点附近 |
| 互换性 | 优先选择具有互换性的质控品 |

**b) 浓度水平设置**

⚠️ 示例，请替换：

| 检验项目类型 | 推荐浓度水平 | 说明 |
|------------|------------|------|
| 临床化学 | 低值 + 高值 (2 水平) | 覆盖临床决策水平 |
| 血常规 | 低值 + 正常值 + 高值 (3 水平) | - |
| 凝血功能 | 正常值 + 异常值 (2 水平) | - |
| 免疫学 | 阴性 + 弱阳性 + 阳性 (3 水平) | 弱阳性覆盖 cut-off 附近 |
| 血气分析 | 低值 + 正常值 + 高值 (3 水平) | - |

**c) 第三方 IQC 材料的考虑**

- 应考虑使用独立于检测试剂制造商的第三方质控品
- 第三方质控品可减少制造商质控品可能存在的偏倚或基质效应掩盖
- 至少对关键项目使用第三方质控品进行补充验证

---

### 5.2 IQC 程序设计 (§7.3.7.2)

**a) IQC 频率**

⚠️ 示例，请替换：

| 检验类型 | IQC 频率 | 说明 |
|---------|---------|------|
| 常规生化 | 每天至少 1 次（每个分析批次） | 开机后、患者样本前 |
| 血常规 | 每天至少 1 次 | 每个工作班次开始前 |
| 凝血功能 | 每天至少 1 次 | 每个分析批次 |
| 免疫学 | 每天至少 1 次 | 每个分析批次 |
| 血气分析 | 每 8 小时至少 1 次 | 24 小时运行项目 |
| POCT | 每天使用前 1 次 | 或按制造商要求 |

**b) 额外执行 IQC 的时机**

- 更换试剂批号后
- 设备维护/维修后
- 校准后
- 环境条件显著变化时
- 出现可疑患者结果时

**c) IQC 规则 — Westgard 规则**

本实验室采用以下 Westgard 多规则质控方案：

| 规则 | 判断标准 | 误差类型 | 处理 |
|------|---------|---------|------|
| 1-2s | 1 个质控值超过 ±2SD | **警告规则** | 不拒绝，启动后续规则检查 |
| 1-3s | 1 个质控值超过 ±3SD | 随机误差 | 拒绝该批次结果 |
| 2-2s | 连续 2 个质控值超过同侧 ±2SD | 系统误差 | 拒绝该批次结果 |
| R-4s | 同一批次内 2 个质控值之差超过 4SD | 随机误差 | 拒绝该批次结果 |
| 4-1s | 连续 4 个质控值超过同侧 ±1SD | 系统误差 | 拒绝该批次结果 |
| 10x | 连续 10 个质控值在均值同一侧 | 系统误差 | 拒绝该批次结果 |

> **注**：各专业组可根据检验项目特点和 Sigma 度量值选择适当的质控规则组合。

### 5.3 替代 IQC 方法 (§7.3.7.2)

> **ISO 15189:2022 原文要求**：当无法使用质量控制材料时，实验室应尽可能使用替代方法来监控检验过程的可靠性。

适用于难以获得质控品的项目或作为常规 IQC 的补充：

| 替代方法 | 适用场景 | 说明 |
|---------|---------|------|
| 患者结果趋势分析 | 所有项目 | 监控患者结果均值/中位数的趋势变化 |
| 移动平均值 (Xbar) | 高通量项目 | 利用一组患者结果计算移动均值 |
| 患者结果重复测定 | 部分项目 | 随机抽取患者样本重复测定 |
| 阴阳性对照 | 定性检验 | 使用已知阴性和阳性标本 |
| 仪器内置质控 | 部分仪器 | 利用仪器自带的电子质控或功能检查 |

---

### 5.4 IQC 数据记录和趋势分析 (§7.3.7.2)

**a) 记录要求**

每次 IQC 运行应记录以下信息：
- 日期和时间
- 质控品批号和浓度水平
- 测定值
- 靶值和控制限
- 判断结果（在控/失控）
- 操作人员
- 失控时的处理措施

记录保存于 LIS 系统或《室内质控原始记录表》(JYK-FM-07-003-01)。

**b) 趋势分析**

- 绘制 Levey-Jennings 质控图，直观展示质控数据的趋势变化
- 关注以下趋势信号：
  - 连续 7 个点在均值同一侧（趋势）
  - 连续 7 个点单方向递增或递减（漂移）
  - 质控数据 CV% 逐渐增大（精密度下降）
- 每月对 IQC 数据进行统计分析，计算月均值、SD、CV%

---

### 5.5 失控处理流程 (§7.3.7.2)

当质控结果判断为失控时，执行以下流程：

```
发现失控
  ↓
1. 暂停发放该批次/该项目的患者结果
  ↓
2. 查找原因（按以下顺序排查）：
   a) 质控品问题（过期、保存不当、未充分混匀）
   b) 操作问题（操作失误、加样量、温度）
   c) 试剂问题（过期、保存不当、批号更换）
   d) 仪器问题（堵塞、光源老化、温控异常）
   e) 校准问题（校准过期、校准品问题）
  ↓
3. 纠正原因后重新做 IQC
  ↓
4. IQC 在控 → 对失控期间已发放的患者结果进行回顾性评估
  ↓
5. 评估是否需要召回/修正已发放结果
  ↓
6. 记录失控原因、纠正措施、患者结果影响评估
```

失控记录于《质控失控处理记录表》(JYK-FM-07-003-02)。

---

### 5.6 IQC 数据定期审查 (§7.3.7.2)

- 每月由各专业组组长审查本组 IQC 数据
- 每季度由质量负责人组织 IQC 数据汇总审查
- 年度 IQC 数据纳入管理评审
- 审查内容：
  - 各项目 IQC 的精密度趋势（CV% 变化）
  - 失控频率和原因分析
  - 质控规则的适用性评估
  - 质控品批号更换时的衔接情况

---

### 5.7 IQC 与临床决策的关联 (§7.3.7.2)

- IQC 浓度水平的设定应考虑临床决策点
- 当 IQC 在临床决策水平附近失控时，应特别关注对临床决策的影响

⚠️ 示例，请替换：

| 检验项目 | 临床决策点 | IQC 覆盖要求 |
|---------|-----------|-------------|
| 葡萄糖 | 3.9 mmol/L（低血糖）、7.0 mmol/L（糖尿病诊断） | 低值质控应覆盖 3.9 附近 |
| HbA1c | 6.5%（糖尿病诊断） | 质控值应覆盖 6.5% 附近 |
| 肌钙蛋白 I | 第 99 百分位 cut-off | 低值质控应覆盖 cut-off 附近 |
| TSH | 0.27 mIU/L（低值）、4.2 mIU/L（高值） | 2 水平质控分别覆盖 |

## 5. 程序内容

### 5.1 EQA 计划选择和登记 (§7.3.7.3)

> **ISO 15189:2022 原文要求**：实验室应参加 EQA 计划，该计划基本上涵盖了实验室提供的检验和检验结果的解释。实验室应监控 EQA 活动的结果，并在结果不符合规定标准时实施纠正措施。

**a) 计划选择原则**

- 优先选择经 CNAS 认可或卫生行政部门组织的 EQA/PT 计划
- 应覆盖本实验室开展的全部检验专业和主要检验项目
- 每个检验项目应至少参加一个 EQA 计划

⚠️ 示例，请替换：

| 专业组 | EQA 计划 | 组织机构 | 频次 |
|--------|---------|---------|------|
| 临床化学 | 临床化学室间质评 | 卫生部临检中心/省临检中心 | 2 次/年 |
| 临床免疫 | 免疫学室间质评 | 卫生部临检中心 | 2 次/年 |
| 临床血液 | 血液学室间质评 | 卫生部临检中心 | 2 次/年 |
| 临床微生物 | 微生物学室间质评 | 卫生部临检中心 | 2 次/年 |
| 凝血功能 | 凝血室间质评 | 卫生部临检中心 | 2 次/年 |

**b) 登记管理**

- 每年初由质量负责人制定年度 EQA 参加计划
- EQA 计划登记信息包括：计划名称、组织机构、检验项目、时间安排、费用
- 登记于《年度 EQA 计划登记表》(JYK-FM-07-004-01)

---

### 5.2 EQA 样本处理 (§7.3.7.3)

> **ISO 15189:2022 原文要求**：EQA 样品应纳入常规工作流程中，由通常执行患者样品检查的人员进行检验，并使用与常规患者样品相同的程序。

**a) 样本接收和保存**

- EQA 样品到达后登记，核查样品完整性
- 按照 EQA 组织方要求保存样品
- 记录接收日期、保存条件

**b) 检测要求**

- EQA 样品必须按照常规患者样本的检测流程处理
- 由日常承担该项目的检验人员执行操作
- 不得对 EQA 样品进行特殊处理或重复检测后选择最佳结果
- 不得与其他实验室交换 EQA 检测结果

**c) 结果提交**

- 在规定时间内提交检测结果
- 提交前由专业组组长核对
- 保留 EQA 检测原始记录

---

### 5.3 EQA 结果评估和趋势分析 (§7.3.7.3)

**a) 结果评估标准**

| 评价指标 | 可接受 | 需关注 | 不可接受 |
|---------|--------|--------|---------|
| Z 分数 | \|Z\| ≤ 2.0 | 2.0 < \|Z\| < 3.0 | \|Z\| ≥ 3.0 |
| PT 评分 | ≥ 80 分 | 60-79 分 | < 60 分 |
| 定性项目 | 与靶值一致 | - | 与靶值不一致 |

**b) 趋势分析**

- 每次 EQA 回报后，对结果进行分析和记录
- 绘制各项目的 EQA 结果趋势图（偏倚趋势）
- 关注持续同方向偏倚（提示系统误差）
- 年度汇总分析，评估各项目 EQA 成绩趋势
- 分析记录于《EQA 结果分析报告》(JYK-FM-07-004-02)

---

### 5.4 EQA 失败处理措施 (§7.3.7.3)

当 EQA 结果不满足评价标准时，执行以下流程：

```
收到 EQA 不合格结果通知
  ↓
1. 立即组织原因分析
   - 样品处理是否正确
   - 检测方法/仪器状态
   - 试剂批号和校准状态
   - IQC 数据是否在控
   - 计算/录入/传输是否有误
  ↓
2. 确定根本原因
  ↓
3. 制定纠正措施
  ↓
4. 实施纠正措施并验证有效性
  ↓
5. 评估对已发放患者结果的影响
  ↓
6. 必要时追溯和通知临床
  ↓
7. 记录全部处理过程
```

EQA 失败处理记录于《EQA 不合格处理记录》(JYK-FM-07-004-03)。

---

### 5.5 替代方案 (§7.3.7.3)

> **ISO 15189:2022 原文要求**：当没有可用的 EQA 计划时，实验室应使用其他方法来评估检验结果的可接受性。

当特定检验项目无 EQA 计划可用时，采取以下替代方案：

| 替代方案 | 说明 | 频率 |
|---------|------|------|
| 实验室间比较 | 与其他实验室交换样品进行比对 | 至少每半年 1 次 |
| 样品交换 | 与使用相同方法的实验室交换患者样品 | 至少每半年 1 次 |
| 有证参考物质检测 | 使用 CRM 评估检测正确度 | 至少每半年 1 次 |
| 患者样本重复检测 | 不同时间对同一样本重复测定 | 根据项目特点确定 |

替代方案的结果应按照 EQA 相同的流程进行评估和记录。

---

### 5.6 比对与可比性管理 (§7.3.7.4)

> **ISO 15189:2022 原文要求**：当同一检验由不同程序、设备或不同地点执行时，应有确定的机制确保结果的可比性。

**a) 适用范围**

- 同一检验项目使用不同仪器（如同型号备用机）
- 同一检验项目使用不同方法/试剂系统
- 不同地点（院区）的同一检验项目

**b) 比对程序**

⚠️ 示例，请替换：

| 比对内容 | 样本要求 | 频率 | 评价标准 |
|---------|---------|------|---------|
| 同型号仪器间比对 | 至少 20 例患者样本，覆盖高中低水平 | 每 【6 个月】 1 次 | 偏差 ≤ 1/2 TEa |
| 不同方法间比对 | 至少 40 例患者样本 | 方法引进时及每 【年】 | 相关系数 r ≥ 0.975，偏差 ≤ TEa |
| 不同院区间比对 | 至少 20 例患者样本 | 每 【6 个月】 1 次 | 偏差 ≤ 1/2 TEa |

**c) 不可比结果的处理**

- 比对结果不满足标准时，应查找原因
- 必要时进行校准调整或方法修正
- 在结果修正前，报告中应标注所使用的仪器/方法
- 比对记录于《仪器/方法比对记录》(JYK-FM-07-004-04)

## 1. 目的

规范本实验室检验后过程的管理，确保检验结果经过适当审核后准确、及时地报告给授权用户，并对检验后的样本进行妥善处理。

适用于本实验室检验后过程的全部活动，包括结果审核和发布、危急值报告、报告要求、结果修正和样本检查后处理。

## 5. 程序内容

### 5.1 结果审核和发布 (§7.4.1)

> **ISO 15189:2022 原文要求**：实验室应有结果审核和发布的文件化程序。授权人员应在发布前审核检验结果。审核应包括对 IQC 数据和适当临床信息的评估。

**a) 结果审核层级**

| 审核层级 | 审核人员 | 审核内容 |
|---------|---------|---------|
| 初审 | 检验人员 | 结果的合理性、仪器状态、IQC 在控状态 |
| 复审 | 专业组组长/高级技师 | 异常结果评估、Delta Check、临床一致性 |
| 特殊审核 | 技术负责人/授权人员 | 危急值、高度异常结果、投诉相关结果 |

**b) 审核内容**

结果审核应至少包含以下检查：
- 当日 IQC 结果是否在控
- 结果是否在分析测量范围内
- 结果与患者诊断信息的一致性
- Delta Check：与患者历史结果的一致性
- 标本质量的影响（如溶血、脂血标注）
- 结果的逻辑一致性（如多项目间的逻辑关系）

---

### 5.2 危急值/关键结果报告 (§7.4.1)

> **ISO 15189:2022 原文要求**：实验室应有文件化的程序来识别和立即通知临床的紧急检验结果（危急值）。

**a) 危急值项目和范围**

⚠️ 示例，请替换：

| 检验项目 | 低危急值 | 高危急值 | 单位 |
|---------|---------|---------|------|
| 葡萄糖 | < 2.8 | > 22.2 | mmol/L |
| 钾 (K⁺) | < 3.0 | > 6.0 | mmol/L |
| 钠 (Na⁺) | < 120 | > 160 | mmol/L |
| 钙 (Ca²⁺) | < 1.75 | > 3.50 | mmol/L |
| 血红蛋白 (Hb) | < 50 | > 200 | g/L |
| 血小板 (PLT) | < 30 | > 1000 | ×10⁹/L |
| 白细胞 (WBC) | < 2.0 | > 30.0 | ×10⁹/L |
| PT-INR | - | > 5.0 | - |
| 肌钙蛋白 I | - | > 【cut-off × 10】 | ng/mL |

**b) 危急值通报流程**

```
检验系统触发危急值
  ↓
1. 检验人员确认结果（必要时复查）
  ↓
2. 专业组组长审核（如在岗）
  ↓
3. 电话通知患者所在科室（15 分钟内）
  ↓
4. 接听人员复述确认
  ↓
5. 记录通知信息（时间、通知人、接收人、复述内容）
  ↓
6. 发放报告并标注危急值标识
```

**c) 记录要求**

危急值通报记录应包含：
- 患者标识和所在科室
- 检验项目和结果值
- 通知时间（精确到分钟）
- 通知人姓名
- 接收人姓名和职称
- 接收人复述结果
- 通知方式（电话/面对面等）

记录于 LIS 系统或《危急值报告登记表》(JYK-FM-07-005-01)。

---

### 5.3 对结果的特殊考虑 (§7.4.1)

以下情况需要特别关注：
- 溶血、脂血、黄疸等标本异常对结果的影响
- 用药干扰（如生物素对免疫检测的干扰）
- 极端结果（超出线性范围需稀释重测）
- 检测方法局限性导致的已知假阳性/假阴性

相关信息应在报告中以备注形式提供。

---

### 5.4 自动选择、审查、发布和报告 (§7.4.1)

> **ISO 15189:2022 原文要求**：当使用自动选择、审查和发布结果的系统时，应有文件化的程序来验证这些系统。

**a) 自动审核规则**

本实验室 LIS 系统设置自动审核规则，符合以下全部条件的结果可自动发布：

⚠️ 示例，请替换：

| 自动审核条件 | 说明 |
|------------|------|
| IQC 当日在控 | 当日 IQC 结果通过 |
| 结果在可报告范围内 | 未超出分析测量范围 |
| 非危急值 | 结果未触发危急值警报 |
| Delta Check 通过 | 与历史结果差异在允许范围内 |
| 非特殊标注标本 | 标本无溶血/脂血/例外接收等标注 |
| 非首次检验项目 | 患者有该项目的历史结果 |

**b) 自动审核规则的验证和维护**

- 自动审核规则在启用前应经过验证
- 至少每 【年/半年】 审查自动审核规则的有效性
- 监控自动审核通过率和人工干预率
- 自动审核规则的修改须经技术负责人批准

---

### 5.5 报告要求 (§7.4.1)

> **ISO 15189:2022 原文要求**：检验报告应包含以下信息（a~m 项）。

检验报告应包含以下全部信息：

| 序号 | 报告内容（ISO 要求） | 说明 |
|------|-------------------|------|
| a | 患者唯一标识 | 姓名、住院号/门诊号、性别、年龄 |
| b | 申请医生/临床科室 | 申请者标识 |
| c | 实验室标识 | 实验室名称、地址、联系方式 |
| d | 检验项目名称 | 标准化名称 |
| e | 检验方法（必要时） | 当方法影响结果解释时标注 |
| f | 标本类型 | 血清/全血/尿液等 |
| g | 采集日期和时间 | - |
| h | 报告日期和时间 | - |
| i | 检验结果 | 包含单位 |
| j | 参考区间 | 适用的参考区间或临床决策极限 |
| k | 结果的解释性备注 | 如溶血影响说明、检测限说明等 |
| l | 审核人标识 | 审核发布人员的姓名/工号 |
| m | 页码标识 | 页码和总页数（多页报告时） |

---

### 5.6 报告附加信息 (§7.4.1)

根据需要，报告中可包含以下附加信息：
- 标本质量状态说明（如：溶血标本，钾结果可能偏高）
- 检测方法变更通知
- 参考区间变更说明
- 检测试剂/仪器信息（用于结果溯源时）
- 检验结果的临床解释建议

---

### 5.7 报告结果的修正 (§7.4.2)

> **ISO 15189:2022 原文要求**：当需要修改已发布的报告时，应保留原始报告和修改记录，修改报告应明确标识为修正报告，并通知相关用户。

**a) 结果修正流程**

```
发现已报告结果有误
  ↓
1. 查明原因（计算错误、转录错误、标本混淆、仪器故障等）
  ↓
2. 出具修正报告
   - 明确标注"修正报告"
   - 包含原始结果和修正后结果
   - 注明修正原因
   - 注明修正日期和时间
  ↓
3. 通知相关临床科室/医生
  ↓
4. 原始报告保留，不得删除或覆盖
  ↓
5. 记录修正事件及通知情况
```

**b) 可追溯性**

- LIS 系统应保留报告的全部修改历史
- 可追溯每次修改的时间、操作人员和修改内容
- 修正记录于《检验报告修正记录》(JYK-FM-07-005-02)

---

### 5.8 样本检查后处理 (§7.4.3)

> **ISO 15189:2022 原文要求**：实验室应有样品在检查后安全处理和保留/储存/丢弃的文件化程序。

**a) 样本保留时间和储存条件**

⚠️ 示例，请替换：

| 标本类型 | 保留时间 | 储存条件 | 备注 |
|---------|---------|---------|------|
| 血清/血浆（常规） | 7 天 | 2-8°C | 特殊项目延长至 30 天 |
| EDTA 全血 | 24 小时 | 室温或 2-8°C | - |
| 尿液常规 | 当日 | 2-8°C | - |
| 微生物培养标本 | 培养完成后 7 天 | 按培养条件 | 阳性株按规定保留 |

- 丢弃记录保留

## 5. 程序内容

### 5.1 不合格工作的识别和分类 (§7.5)

> **ISO 15189:2022 原文要求**：实验室应有识别不合格工作的程序。当检验前、检验中或检验后任何环节的活动或结果不符合其程序或用户的约定要求时，应适用本程序。

**a) 不合格工作分类**

| 分类 | 示例 |
|------|------|
| 检验前不合格 | 标本不合格（溶血、凝块、量不足、标识错误）、申请单信息不全、运输条件不符 |
| 检验中不合格 | IQC 失控、设备故障、试剂过期使用、校准失效、操作失误 |
| 检验后不合格 | 结果报告错误、危急值通报延迟、报告延迟、结果转录错误 |
| 外部投诉 | 临床科室投诉、患者投诉 |

**b) 严重程度分级**

| 等级 | 定义 | 示例 |
|------|------|------|
| 严重 (A 级) | 可能对患者造成伤害或已造成伤害 | 标本混淆导致错误报告、危急值未通报 |
| 较重 (B 级) | 可能影响检验结果但尚未对患者造成伤害 | IQC 失控期间已发放结果、试剂过期使用 |
| 一般 (C 级) | 偏离程序但对结果无实质影响 | 记录不完整、报告格式不规范 |

---

### 5.2 风险分析流程 (§7.5)

> **ISO 15189:2022 原文要求**：当确定不合格工作可能再次发生，或对实验室与其程序的符合性存有怀疑时，实验室应采取纠正措施。

**a) 即时措施**

发现不合格工作后，应立即采取以下控制措施：

| 不合格情形 | 即时措施 |
|-----------|---------|
| IQC 失控 | 暂停发放该项目患者结果 |
| 设备故障 | 停止使用该设备，启用备用方案 |
| 标本不合格 | 拒收或标注后按例外处理流程 |
| 报告错误 | 立即通知临床，出具修正报告 |
| 危急值通报延迟 | 立即补充通报 |

**b) 长期措施（根本原因分析和纠正措施）**

```
不合格事件记录
  ↓
1. 根本原因分析
   - 使用"5 Why"分析法或鱼骨图
   - 识别系统性因素（人员、设备、方法、环境、材料）
  ↓
2. 制定纠正措施
   - 措施应针对根本原因
   - 确定责任人和完成时限
  ↓
3. 实施纠正措施
  ↓
4. 验证纠正措施的有效性
  ↓
5. 必要时修订相关文件/程序
```

---

### 5.3 对患者有伤害风险时的处理 (§7.5)

> **ISO 15189:2022 原文要求**：当确定不合格工作可能已经或可能对患者造成伤害时，实验室应立即停止检查、扣留报告。

当评估认为不合格工作可能对患者造成伤害时：

| 步骤 | 措施 | 说明 |
|------|------|------|
| 1 | 立即停止相关检查 | 暂停该项目/该设备的检验活动 |
| 2 | 扣留未发放的报告 | 禁止发放可能受影响的结果 |
| 3 | 通知科室负责人和质量负责人 | 第一时间上报 |
| 4 | 评估受影响的范围 | 追溯受影响的时间段和患者数量 |
| 5 | 通知临床相关科室 | 告知可能受影响的结果和建议措施 |
| 6 | 必要时向医院管理层和主管部门报告 | 重大事件上报 |

---

### 5.4 临床意义评估 (§7.5)

对于已发放的不合格结果，应评估其临床意义：

- 错误结果是否可能影响临床诊断或治疗决策
- 错误结果的偏差方向和程度
- 患者是否已基于错误结果接受了治疗
- 是否需要建议临床对患者进行额外检查或监测

临床意义评估应由技术负责人主导，必要时与临床医生协商。

---

### 5.5 可接受性决定 (§7.5)

不合格工作的处理决定包括：

| 决定 | 适用条件 | 审批权限 |
|------|---------|---------|
| 按原样接受 | 不合格对结果无实质影响 | 专业组组长 |
| 例外接受 | 有条件接受（如标本不可替代） | 技术负责人 |
| 拒绝/作废 | 不合格影响结果可靠性 | 专业组组长 |
| 返工/重检 | 可通过重新检测获得可靠结果 | 检验人员 |

---

### 5.6 结果修改和用户通知 (§7.5)

- 已发放的不合格结果应按照《检验后过程管理程序》(JYK-SOP-07-005) 第 5.7 节执行修正
- 通知临床用户时应说明：
  - 受影响的检验项目和结果
  - 不合格的原因
  - 修正后的结果（如适用）
  - 对患者的建议措施（如需重新采样检测）
- 通知记录保存

---

### 5.7 恢复工作的批准 (§7.5)

不合格事件处理后，恢复检验工作需满足以下条件：

| 恢复条件 | 验证方式 |
|---------|---------|
| 不合格原因已消除 | 根本原因分析完成 |
| 纠正措施已实施 | 纠正措施执行记录 |
| 检验系统状态正常 | IQC 通过、设备性能验证通过 |
| 经授权人员批准 | 质量负责人或技术负责人签字 |

---

### 5.8 纠正措施（与 §8.7 联动）(§7.5)

> **ISO 15189:2022 原文要求**：实验室应消除已识别的不符合的原因以防止再次发生（§8.7）。

**a) 纠正措施触发条件**

- A 级和 B 级不合格事件
- 同类不合格事件重复发生
- EQA/PT 不合格
- 内部审核发现的不符合
- 管理评审中识别的问题

**b) 纠正措施流程**

- 根本原因分析 → 制定纠正措施 → 实施 → 验证有效性 → 关闭
- 纠正措施记录于《纠正措施记录表》(JYK-FM-07-006-02)
- 纠正措施的有效性应在规定时限内验证
- 无效的纠正措施应重新分析原因并制定新措施

**c) 数据分析**

- 定期汇总不合格工作数据，分析趋势
- 识别不合格工作的高发环节和类型
- 为质量改进提供数据支持
- 不合格工作统计纳入管理评审

## 5. 程序内容

### 5.1 信息管理权限和责任 (§7.6)

> **ISO 15189:2022 原文要求**：实验室应有管理信息的文件化程序，确保信息的收集、处理、记录、报告、保留和检索的完整性和保密性。

**a) 权限架构**

| 权限级别 | 角色 | 系统权限范围 |
|---------|------|------------|
| 系统管理员 | 信息管理员 | 用户管理、系统配置、数据备份、审计日志查看 |
| 高级用户 | 科室负责人、技术负责人、质量负责人 | 结果审核发布、参数修改、统计查询、报告修正 |
| 普通用户 | 检验人员 | 样本接收、结果录入/传输、IQC 操作、结果初审 |
| 查询用户 | 临床医生、护士 | 检验结果查询和打印 |

**b) 权限管理原则**

- 最小权限原则：用户只获得完成工作所需的最低权限
- 职责分离：关键操作（如结果修正）需要不同人员执行和审批
- 定期审查：至少每 【年/半年】 审查用户权限的适当性
- 离岗人员应及时停用账号

**c) 保密要求**

- 所有系统用户应签署保密协议
- 患者信息仅限授权人员访问
- 禁止向未授权人员泄露患者检验数据
- 数据使用应符合《个人信息保护法》等法律法规要求

### 5.2 信息系统管理 (§7.6)

> **ISO 15189:2022 原文要求**：用于收集、处理、记录、报告、存储或检索检验数据和信息的信息系统应经过验证和确认。实验室应确保数据的完整性和信息系统的网络安全。

#### 5.2.1 系统验证和确认

信息系统在以下情况下应进行验证/确认：

| 触发条件 | 验证/确认内容 |
|---------|-------------|
| 新 LIS 系统上线 | 全面功能验证和性能确认 |
| 系统重大升级 | 升级功能和关联功能的验证 |
| 接口变更 | 仪器-LIS 接口数据传输验证 |
| 服务器迁移 | 数据完整性和系统功能验证 |

验证内容至少包括：

⚠️ 示例，请替换：

| 验证项目 | 验证方法 | 接受标准 |
|---------|---------|---------|
| 患者信息传输 | 录入测试数据，核对 LIS 显示 | 100% 一致 |
| 仪器结果传输 | 对比仪器原始数据与 LIS 接收数据 | 100% 一致 |
| 结果计算 | 已知值验算 | 计算结果 100% 正确 |
| 参考区间标记 | 已知异常值测试 | 标记正确率 100% |
| 危急值警报 | 模拟危急值数据 | 正确触发报警 |
| 报告打印 | 打印测试报告 | 内容完整、格式正确 |
| 审计追踪 | 执行修改操作后查看日志 | 记录完整、准确 |

验证记录于《LIS 系统验证报告》(JYK-FM-07-007-01)。

#### 5.2.2 授权用户管理

- 新用户账号由信息管理员根据科室负责人批准创建
- 每个用户使用唯一的用户名和密码
- 密码策略：
  - 最小长度 【8】 位
  - 包含字母和数字
  - 定期更换（每 【90】 天）
  - 连续 【5】 次输错密码自动锁定

#### 5.2.3 网络安全

- LIS 系统部署在医院内网，与外网隔离
- 安装防病毒软件并定期更新
- 系统补丁及时更新
- USB 端口管理：禁止未授权的 USB 设备接入
- 网络访问日志定期审查
- 与医院信息科协同进行安全评估

#### 5.2.4 数据完整性

- LIS 系统应具有审计追踪功能
- 所有数据的创建、修改、删除操作均应自动记录
- 审计记录包含：操作人员、操作时间、操作内容（修改前/后值）
- 审计记录不可被修改或删除
- 电子签名具有法律等效性

|---------|------|--------|

- 用户权限变更

变更由技术负责人审批，变更前后进行验证，记录于《LIS 系统变更记录》(JYK-FM-07-007-02)。

### 5.4 停工计划 (§7.6)

> **ISO 15189:2022 原文要求**：实验室应有计划和文件化的程序来应对信息系统故障，包括维持检验服务的安排。

**a) 信息系统故障分级**

| 级别 | 故障描述 | 响应时间 |
|------|---------|---------|
| 1 级（紧急） | LIS 系统完全不可用 | 立即启动应急方案 |
| 2 级（严重） | 部分功能不可用（如打印故障、接口中断） | 1 小时内处理 |
| 3 级（一般） | 性能下降但不影响基本功能 | 工作日内处理 |

**b) 应急方案（LIS 完全不可用时）**

```
发现 LIS 系统故障
  ↓
1. 立即通知信息管理员和医院信息科
  ↓
2. 启动手工操作应急方案：
   - 使用纸质申请单和手工登记
   - 使用纸质记录本记录检验结果
   - 手工计算和审核结果
   - 使用纸质报告单发放结果
   - 危急值仍按正常流程电话通报
  ↓
3. 故障修复后：
   - 将手工记录的数据补录入 LIS 系统
   - 核对补录数据的完整性和准确性
   - 验证 LIS 系统功能恢复正常
```

**c) 应急物资准备**

- 备用纸质申请单
- 手工结果记录本
- 纸质报告单模板
- 手工计算工具（计算器）
- 应急联系人名单

应急方案每 【年】 进行一次演练，演练记录于《停工应急演练记录》(JYK-FM-07-007-03)。

---

### 5.5 场外管理 (§7.6)

> **ISO 15189:2022 原文要求**：当信息系统由外部提供商管理时，实验室应确保提供商满足本程序的要求。

当 LIS 系统或相关信息系统由外部供应商维护时：

| 管理要求 | 说明 |
|---------|------|
| 服务协议 | 签订服务水平协议 (SLA)，明确响应时间、可用性指标 |
| 保密协议 | 供应商签署保密协议，保护患者数据 |
| 访问控制 | 供应商远程访问需经审批，限定访问范围和时间 |
| 变更通知 | 供应商系统变更应提前通知实验室 |
| 数据所有权 | 明确数据归实验室所有 |
| 数据位置 | 确认数据存储位置符合法规要求 |
| 定期评估 | 每 【年】 评估供应商的服务质量和安全合规性 |

### 5.6 数据备份和恢复 (§7.6)

**a) 备份策略**

⚠️ 示例，请替换：

**b) 备份验证**

```
数据丢失/损坏事件

  ↓

  ↓

  ↓
6. 记录恢复过程和结果
```

| EQA 数据 | 长期 | 体系要求 |
| 系统日志 | 【至少 1 年】 | 安全审计要求 |
| 审计追踪记录 | 与相关数据等同 | 数据完整性要求 |

# 生成规则（强制）

你是实验医学科 ISO 15189:2022 质量管理体系文件专家。生成任何文件时，必须遵循以下全部规则。

## 2. 编号与层级

- 标题层级：一级"1"，二级"1.1"，三级"1.1.1"，最深不超过四级
- 文件编号体系：QM（质量手册）、PER（人员）、DOC（文件记录）、QUA（质量）、SAF（安全）、ANA（分析过程）、INS（设备物资）、MET（方法性能）
- 记录表格编号：[前缀]-REC-[序号]

## 3. 标准引用

- 必须引用 ISO 15189:2022 对应条款，格式：ISO 15189:2022 第 X.X.X 条
- 必须同时引用 CNAS-CL02:2023 对应条款
- 每个程序要求须追溯到至少一个标准条款

## 4. 语言风格

- 中文为主，专业术语保留英文原文并附中文
- 操作步骤使用祈使句（"应""必须""shall"）
- 禁止模糊表述。生成内容中出现以下任何词汇，必须立即替换为右侧的精确表述：

| 禁止用法 | 必须替换为 |
|----------|-----------|
| 大约/约/左右/差不多 | 具体数值或范围（如"2-8℃""30±2分钟"） |
| 一般来说/通常/往往 | 删除修饰词直接陈述；有例外用"除…外"列出 |
| 及时/尽快/尽早 | 明确时限（如"2小时内""当日完成""发现后30分钟内"） |
| 相关人员/有关部门 | 具体岗位（如"质量负责人""检验科主任""专业组长"） |
| 适当/合理/必要时 | 触发条件（如"当温度偏差>2℃时""质控违反Westgard规则时"） |
| 定期/经常/不定期 | 频率（如"每月1次""每季度至少1次""每6个月"） |
| 做好记录/注意保存 | 指向表格（如"填写PER-REC-003""记录于INS-REC-005"） |

## 5. 术语规范

必须使用 ISO 15189:2022 标准术语，禁止使用旧术语：

| 禁止使用 | 必须改为 |
|----------|---------|

| 分析过程/分析阶段 | 检验过程 |
| 分析后过程/分析后阶段 | 检验后过程 |
| 不确定度（单独使用） | 测量不确定度 |
| 正常值范围/正常参考值 | 生物参考区间 |
| 标本 | 样品 |
| 检验项目 | 检测项目 |
| 准确度 | 正确度 |
| 质控品 | 质控物 |
| 线性范围 | 可报告范围 |

- 新建文件版本号：第 A 版第 0 次修订
