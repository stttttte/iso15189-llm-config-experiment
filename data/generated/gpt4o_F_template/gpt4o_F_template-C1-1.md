# 内部质量控制程序文件审查报告

## 一、基本信息

- **受审文件名称**：内部质量控制程序文件  
- **程序编号**：ANA-003  
- **版本/生效日期**：1.5 / 2024-12-01  
- **审查依据**：  
  1. **ISO 15189:2022**  
  2. **CNAS-CL02:2023《医学实验室 质量和能力认可准则》**  
  3. **CNAS-CL02-A001 等相关应用说明**（如适用）  
  4. 医学实验室内部质量控制通行要求及 Westgard 多规则应用原则  
- **审查方式**：文件评审  
- **审查结论概述**：该程序文件具备内部质控管理的基本框架，但**标准条款覆盖不完整、术语使用不规范、内容可操作性不足、与相关程序接口不清、章节结构和版本控制存在明显缺陷**。按 CNAS 文件符合性要求判断，**该文件目前不建议直接作为受控正式版投入认可体系运行**，应完成修订后再发布。

---

## 二、总体评价

该文件已涉及以下内容：
- 内部质量控制基本目的和适用范围；
- L-J 图及部分 Westgard 规则；
- 质控频率；
- 异常结果处理；
- 室间质评/能力验证关联；
- 质控记录保存和质量目标。

但从认可评审角度，主要存在以下共性问题：

1. **引用条款不准确、覆盖不完整**：将内部质量控制仅对应到 7.3.1、7.3.2，未体现与结果有效性监控、质量保证、能力验证、测量不确定度、风险控制、不合格工作等相关条款的系统关联。
2. **程序性不足**：多处只列原则，缺乏“谁做、何时做、怎么做、形成什么记录、异常时如何升级处理”的可执行要求。
3. **文件规范性较差**：版本修订、章节编号、附录缺失、定义缺失等，反映文件控制意识不足。
4. **技术要求表达不严谨**：如质控水平设置、规则适用条件、室间质评频率表述等存在不当或矛盾。
5. **与其他体系文件接口不清**：如不合格工作、纠正措施、能力验证、报告召回、风险管理等衔接不完整。

## 三、问题分级原则说明

本次审查采用以下分级逻辑：

- **严重不符合 / 重大问题**：可能导致标准关键要求缺失，影响结果有效性控制，或可能影响患者安全、认可符合性。
- **一般不符合 / 重要问题**：虽不构成体系失效，但条款覆盖不充分、程序不完整，影响执行有效性。
- **观察项 / 一般问题**：格式、术语、表述、文件规范性问题，虽不必然直接影响结果，但反映管理薄弱，建议整改。

## 四、逐项审查意见及分级

## 1. 文件结构与版本控制问题

### 1.1 缺少规范修订记录表
- **问题描述**：文件仅列“主要修改记录”，但未设置标准化的修订记录/版本历史表，未体现修订章节、修订原因、批准信息。

- **风险**：不利于追溯版本变更的合规性与受控状态。
- **分级**：**一般不符合**

  - 明确受控版本状态及替代关系。

### 1.2 章节编号不连续，反映文件控制失效
- **问题描述**：第 3 条直接从 3.4 开始，且文中已提示“跳过 3.1-3.2”；章节体系明显不完整。
- **涉及条款**：ISO 15189:2022 第 8.3
- **风险**：提示文件经多次修改后未进行结构性校核，可能存在内容删改遗漏，影响执行和审核。
- **分级**：**一般不符合**
- **建议**：
  - 重新梳理目录和章节编号；
  - 修订时同步执行文件完整性审查；

### 1.3 附录声明存在但实际缺失
- **问题描述**：附录注明应包括 L-J 图模板、质控原始记录表等，但实际未附。
- **涉及条款**：ISO 15189:2022 第 8.3、7.3
- **风险**：程序与记录载体脱节，导致执行无法落地。
- **分级**：**一般不符合**

## 2. 条款引用与覆盖问题

### 2.1 条款引用不充分，未建立与“结果有效性保证”要求的完整对应
- **问题描述**：文件仅声明符合 ISO 15189:2022 第 7.3.1 和 7.3.2 条，但内部质控文件本质上应与**检验结果有效性保证**的整体要求相对应，不应仅局限于局部条款。
- **主要缺陷**：
  - 未体现对**定性/半定量项目结果有效性监控**的安排；
  - 未明确与**能力验证/替代比对**、**不合格工作管理**、**纠正措施**、**结果回顾与再发布**等的接口。
- **分级**：**重要问题**
- **建议**：
  - 重新核对本程序适用标准条款，建立“条款—程序条文—记录”对应表；
  - 明确内部质控、室内比对、室间质评、趋势分析、不合格处理是同一结果有效性保证链条的一部分。

### 2.2 关于“7.3.4 测量不确定度”被纳入本程序缺失项的判断需修正，但相关要求仍应有接口
- **评审意见**：用户给出的“问题 4”认为本程序遗漏 7.3.4 测量不确定度。  
  从主任评审员角度，**该判断部分成立但表述不够准确**：
  - **测量不确定度不一定必须在“内部质量控制程序”中详细展开**；
  - 但该程序至少应说明：内部质控数据、精密度数据、长期性能数据可作为测量不确定度评估的重要输入之一，或引用专门的《测量不确定度评估程序》。
- **分级**：**观察项偏重要**
- **建议**：
  - 若实验室已有独立 MU 程序，则在本文件中增加接口条款；
  - 若无独立程序，则应补充测量不确定度评估职责、数据来源和应用原则。

## 3. 适用范围界定不充分

### 3.1 直接排除定性和半定量项目，不符合结果有效性控制思路
- **问题描述**：文件声明“不适用于定性检验项目和半定量项目”。  
- **评审意见**：这一定义对“内部质控程序”而言过于狭窄。ISO 15189 强调的是**所有检验活动结果有效性保证**。定性与半定量项目同样需要质量控制，只是控制方式不同，例如：
  - 阳性/阴性质控品；
  - 临界值一致性检查；
  - 批内/批间符合率；
  - 人员间比对；
  - 试剂批号验证等。
- **风险**：导致定性项目完全游离于质量保证体系之外。
- **分级**：**严重不符合**
- **建议**：
  - 本程序改为覆盖所有检验项目；
  - 对不同类型项目分章规定：定量、定性、半定量、镜检/形态学项目；
  - 对暂不适用项目说明采用何种替代控制措施，而非直接排除。

## 4. 内部质控技术要求不充分或不严谨

### 4.1 质控规则列示不完整且命名不规范
- **问题描述**：
  - 使用“Levy-Jennings 图”，标准写法应为 **Levey-Jennings**；
  - 使用“1:3s、2:2s、R:4s、10:x̄”等表示法，建议统一为更常见、规范的 **1₃s、2₂s、R₄s、10x** 表述，或在定义中说明。
- **风险**：影响文件专业性与培训一致性。
- **分级**：**观察项**
- **建议**：
  - 增设“术语和定义/缩略语”章节；
  - 统一规则命名、图表名称、判定语义。

### 4.2 Westgard 规则应用条件和操作步骤缺失，影响可执行性
- **问题描述**：虽列出若干规则，但未说明：
  - 单水平还是多水平质控时如何判读；
  - 批内规则与批间规则分别如何应用；
  - 何时属于“警告规则”，何时属于“失控拒绝规则”；
  - 不同项目是否全部适用同一规则组合；
  - 触发后复测几次、校准与否、是否需更换试剂/重新开批等。
- **分级**：**重要问题**
- **建议**：
  - 对每条规则补充：定义、判定场景、处理流程、是否拒绝该批结果；
  - 明确项目分层管理：如生化、免疫、血液学可采用不同规则组合；
  - 增加典型案例和流程图。

### 4.3 质控频率规定存在“一刀切”倾向，缺乏风险基础
- **问题描述**：
  - 规定“8 小时班次每 4 小时 1 次质控”，但未说明是否适用于随机急诊、连续加载、批量检测、低通量项目、换班后首批等情形；
  - “每个检验项目开始分析前，至少进行 1 个水平质控”，与后文“生化和免疫项目应进行高、中、低三个浓度水平质控”之间存在层级不清。
- **评审意见**：CNAS 更关注质控安排是否基于**检验系统稳定性、患者风险、检测批次、厂家说明、项目特性、历史性能**制定。
- **分级**：**重要问题**
- **建议**：
  - 按项目或平台制定质控计划；
  - 明确最少质控水平数、每批/每日/每班次/换瓶/换批/维护后的要求；
  - 对低频项目、急诊项目、POCT 项目制定差异化策略。

### 4.4 未规定质控靶值、标准差、允许范围的建立与审核方法
- **问题描述**：文件只说画 L-J 图和执行规则，但未说明：
  - 靶值和 SD 来源；
  - 新批号质控物靶值建立；
  - 厂家靶值何时可用、何时必须实验室重建；
  - 审核周期；
  - 靶值漂移处理。
- **分级**：**严重不符合**
- **风险**：没有合法有效的统计基础，Westgard 规则无法正确应用。
- **建议**：
  - 补充新质控品启用前的靶值建立程序；
  - 明确至少 20 次或基于规定条件建立均值/SD；
  - 规定不同平台、不同批号分别建图；
  - 明确审核批准权限。

### 4.5 未明确质控失控后的受影响样本范围界定方法
- **问题描述**：第 6.2 仅写“对之前分析的标本进行重新检验”，未说明“之前”指从哪一时点起。
- **分级**：**严重不符合**
- **风险**：无法有效追溯受影响患者结果，直接影响患者安全。
- **建议**：
  - 明确回溯起点：上一次在控点之后、最近一次合格质控之后、某项变更事件之后等；
  - 规定已发报告的召回/更正流程；
  - 与不合格工作和报告更正程序联动。

## 5. 异常处理与不合格工作接口不足

### 5.1 “异常结果”定义混乱，混淆质控异常与患者结果异常
- **问题描述**：将“质控未通过的结果”与“逻辑判定异常的结果”并列为“异常结果”。
- **评审意见**：应至少区分：
  1. **质控异常/检验系统失控**；
  2. **患者结果异常但检验系统可能在控**；
  3. **样本异常（溶血、脂血、凝块等）**；
  4. **临床逻辑不一致需复核的结果**。
- **分级**：**重要问题**
- **建议**：重构异常处理章节，分别规定不同触发条件和处置路径。

### 5.2 质控失控后的处置流程不完整
- **问题描述**：仅规定暂停分析、原因调查、纠正、重新验证，但未包括：
  - 是否停止报告发放；
  - 是否冻结 LIS 审核权限；
  - 是否通知临床；
  - 是否启动不合格工作程序；
  - 是否进行风险评估。
- **分级**：**严重不符合**
- **建议**：
  - 增加“失控升级流程”；
  - 明确一般失控与重大失控的上报层级；
  - 与《不合格工作管理程序》《纠正措施程序》建立交叉引用。

### 5.3 对“纠正措施”和“纠正”混用，缺乏根因分析要求
- **问题描述**：文件写“原因调查”“纠正措施”，但未要求根因分析、有效性验证、关闭标准。
- **分级**：**一般不符合**
- **建议**：增加 CAPA 接口，明确：
  - 何种情形仅需纠正；
  - 何种情形需启动纠正措施；
  - 如何验证有效性。

## 6. 室间质评/能力验证章节问题

### 6.1 室间质评频率规定与实际能力验证安排矛盾
- **问题描述**：文件规定“每月进行一次室间质评”，同时又说“按 ANA-004 能力验证计划执行”，二者逻辑不一致。
- **分级**：**重要问题**
- **建议**：
  - 区分以下概念：
    1. **外部室间质量评价/能力验证（EQA/PT）**：按提供者计划参加；
    2. **实验室间比对**：如无 PT 时实施；
    3. **内部替代评价**：如留样比对、方法学比对、分样复检等。
  - 删除不符合实际的“每月一次”刚性要求，改为“按年度计划参加；对无 EQA/PT 项目制定替代措施”。

### 6.2 内部质控程序中纳入 EQA/PT 但界面不清
- **问题描述**：第 7 条内容实质属于另一程序 ANA-004 管辖，但本程序未明确两者边界。
- **风险**：职责重复或空转。
- **分级**：**一般不符合**
- **建议**：
  - 在本程序中仅说明“结果有效性监控包括 IQC 与 EQA/PT，EQA/PT 详见 ANA-004”；
  - 删除重复操作性内容，保留接口要求。

## 7. 术语和标准一致性问题

### 7.1 “分析过程”与“检验过程”用语不统一
- **问题描述**：第 6.2 使用“分析过程”，与现行标准和程序体系建议用语“检验过程”不一致。
- **评审意见**：该问题存在，但通常属于**术语规范性问题**，除非体系内广泛混用导致职责边界不清。
- **分级**：**观察项**
- **建议**：全文术语统一，并与质量手册术语章节保持一致。

### 7.2 “标本”与“样本”混用
- **问题描述**：第 6.2 使用“标本重新检验”。
- **分级**：**观察项**
- **建议**：统一使用实验室体系规定术语；若历史文件保留“标本”，需在定义中说明同义关系并逐步规范。

### 7.3 缺少“定义和缩略语”章节
- **问题描述**：L-J 图、Westgard、多规则、失控、在控、批、水平等未定义。
- **分级**：**一般不符合**
- **建议**：增加“术语和定义/缩略语”章节。

## 8. 记录、数据管理与质量目标问题

### 8.1 记录要求不充分，缺少审核/复核痕迹
- **问题描述**：记录内容仅列记录人签名，未要求审核人/复核人、判定结论、处置结果。
- **分级**：**一般不符合**
- **建议**：
  - 增加：审核人、审核时间、质控判定、异常编号、纠正措施编号、恢复运行批准人。

### 8.2 数据保存期限写“≥6年”，但缺少依据及载体管理要求
- **问题描述**：虽规定保存期限，但未说明纸质/电子记录、备份、可追溯性、LIS/仪器接口原始数据保存要求。
- **分级**：**观察项**
- **建议**：与记录控制程序联动，明确原始数据、图表、统计报表、备份策略。

### 8.3 质量目标设置欠合理，缺乏定义和统计口径
- **问题描述**：
  - “质控合格率 ≥99%”未说明分母是什么，是项目次数、批次数还是天数；
  - “样本重测率 ≤2%”并不必然反映质量优劣，且受项目性质影响较大；
  - “异常处理及时率 100%”过于绝对，未定义“及时”的判定时限和口径。
- **分级**：**一般不符合**
- **建议**：
  - 将目标转化为可测量、可统计、可解释的指标；
  - 例如：失控事件按规定时限关闭率、质控审核及时率、重大失控事件数、纠正措施有效率等。

## 9. 职责缺失

### 9.1 文件未明确职责分工
- **问题描述**：该程序没有单独规定实验室主任、质量负责人、技术负责人、专业组长、检验人员在内部质控中的职责。
- **风险**：程序无法明确执行主体和审核责任。
- **分级**：**严重不符合**
- **建议**：至少明确：
  - 技术负责人：批准质控方案、审核异常趋势；
  - 质量负责人：监督执行、统计分析、推动改进；
  - 组长：日常审核、失控处置、培训；
  - 检验人员：执行质控、记录、初步判定、上报；
  - 授权签字人/审核人：在失控影响患者结果时的处置责任。
