## 不符合项一：文件评审逾期

### 1. 不符合事实描述

在本次内审中，审核组对文件控制程序执行情况进行检查，发现以下问题：

实验室《文件控制程序》（文件编号：SOP-QM-003，第3版）规定"所有受控文件应每两年进行一次定期评审，由文件起草部门负责组织，质量管理科负责跟踪督促"。经核查文件评审记录台账，发现以下文件超过规定评审周期仍未完成评审：

| 文件编号 | 文件名称 | 上次评审日期 | 应评审日期 | 逾期天数 |
|----------|---------|------------|-----------|---------|
| SOP-BIO-012 | 《血培养标本采集与接种操作规程》 | 2023-09-15 | 2025-09-15 | 201天 |
| SOP-CHM-008 | 《离子色谱法检测电解质操作规程》 | 2023-11-20 | 2025-11-20 | 135天 |
| SOP-HEM-005 | 《凝血功能检测标准操作规程》 | 2024-01-10 | 2026-01-10 | 84天 |

上述3份文件均已超过评审期限，其中最长逾期达201天。文件管理系统中无相关评审提醒记录，质量管理科未对逾期文件进行催办。

**不符合条款**：ISO 15189:2022 第8.3.2条"文件控制——实验室应按规定的时间间隔评审文件，必要时进行修订，确保文件持续适用"。

**不符合等级**：一般不符合

### 2. 原因分析

采用鱼骨图法从人、机、料、法、环五个维度进行根本原因分析：

- **人员因素**：文件管理员岗位于2025年6月发生人员变动，新任管理员未完成系统性培训，对文件评审周期跟踪职责认知不足。各专业组文件起草人未主动关注本组文件评审到期情况。
- **系统因素**：现有LIMS文件管理模块缺乏自动到期预警功能，文件评审跟踪完全依赖人工台账登记，存在遗漏风险。
- **制度因素**：《文件控制程序》中仅规定"质量管理科负责跟踪督促"，但未明确具体的督促频次、方式以及逾期未评审的升级上报机制。

**根本原因**：文件评审到期跟踪机制依赖人工管理，缺乏系统化预警手段，且制度中未建立逾期升级处理流程。

### 3. 纠正措施

#### 3.1 立即纠正（完成期限：2026年4月18日）

对上述3份逾期文件立即启动评审程序。各专业组组长组织本组人员对文件的技术内容、引用标准、操作步骤进行逐条审查，确认是否需要修订。评审完成后填写《文件评审记录表》，经质量负责人批准后归档。

#### 3.2 系统改进（完成期限：2026年5月30日）

**（1）修改文件管理模块增加自动预警功能**

在LIMS系统文件管理模块中增加评审到期自动提醒机制。以下为需修改的后端代码片段：

**文件路径**：`backend/app/services/document_service.py`

```python
# === 新增：文件评审到期预警服务 ===
from datetime import datetime, timedelta
from sqlalchemy import select, and_
from app.models.document import Document
from app.core.database import get_session

async def get_review_overdue_documents(advance_days: int = 30):
    """
    获取即将到期和已逾期的文件列表。
    advance_days: 提前预警天数，默认30天。
    """
    async with get_session() as session:
        now = datetime.now()
        warning_date = now + timedelta(days=advance_days)

        stmt = select(Document).where(
            and_(
                Document.status == "已发布",
                Document.next_review_date <= warning_date,
                Document.is_deleted == False
            )
        ).order_by(Document.next_review_date.asc())

        result = await session.execute(stmt)
        documents = result.scalars().all()

        overdue_list = []
        for doc in documents:
            overdue_list.append({
                "doc_id": doc.id,
                "doc_number": doc.document_number,
                "doc_title": doc.title,
                "next_review_date": doc.next_review_date.isoformat(),
                "overdue_days": (now - doc.next_review_date).days
                    if now > doc.next_review_date else 0,
                "status": "已逾期" if now > doc.next_review_date else "即将到期",
                "responsible_group": doc.group_name
            })
        return overdue_list
```

## 不符合项二：危急值审核缺失

### 1. 不符合事实描述

审核组对检验后过程进行检查时发现，实验室《危急值报告制度》（文件编号：SOP-QA-019，第2版）规定"所有危急值结果在报告临床前，必须经由当班高级技师或组长进行二级审核确认，审核记录应包括审核人签名、审核时间及确认意见"。

抽查2026年1月至3月期间的危急值报告登记台账及LIMS系统审核记录，发现以下问题：

- 2026年1月15日，患者张某某（住院号H20260089），血钾检测结果为2.1 mmol/L（危急值界限：<2.5 mmol/L），LIMS系统中仅有检测技师王某的一级审核记录，缺少二级审核人签字及审核时间。
- 2026年2月22日，患者李某某（住院号H20260215），血小板计数结果为18×10⁹/L（危急值界限：<30×10⁹/L），纸质台账中记录了电话通知临床的时间，但LIMS系统中二级审核字段为空。
- 2026年3月8日，患者赵某某（门诊号M20260412），血糖结果为1.8 mmol/L（危急值界限：<2.8 mmol/L），系统显示二级审核由实习生陈某完成，不符合"高级技师或组长"的资质要求。

上述3例危急值报告均存在二级审核环节缺失或不合规的情况，涉及患者安全风险。

**不符合条款**：ISO 15189:2022 第7.4.1.2条"检验结果的审查——实验室应规定检验结果发布前的审查程序，包括对危急值/警告值结果的审查确认"。

**不符合等级**：严重不符合

### 2. 原因分析

- **系统因素**：LIMS系统在危急值结果触发后，未强制要求完成二级审核即可进行结果发布操作。系统权限设置未限制不具备审核资质的人员进行二级审核操作。
- **人员因素**：部分夜班及节假日值班期间，仅有一名技师在岗，无法满足二级审核的人员配置要求，技师为及时通知临床而跳过审核环节。
- **制度因素**：《危急值报告制度》中未明确夜班/单人值班场景下的替代审核流程，未规定远程审核等应急措施。

**根本原因**：LIMS系统未将危急值二级审核设置为结果发布的强制前置条件，且制度未覆盖特殊值班场景的审核安排。

### 3. 纠正措施

#### 3.1 立即纠正（完成期限：2026年4月11日）

对上述3例危急值报告进行回顾性补充审核。由各专业组组长对原始检测数据、质控状态、仪器运行日志进行追溯核查，确认检测结果准确性，并在LIMS系统中补录二级审核记录，注明"回顾性审核"标识。

# === 修改：危急值结果发布前强制二级审核校验 ===
from app.models.user import User, UserRole
from fastapi import HTTPException, status

CRITICAL_VALUE_REVIEWER_ROLES = ["senior_technician", "group_leader", "lab_director"]

async def validate_critical_value_review(
    examination_id: int,
    reviewer_user_id: int,
    session
) -> bool:
    """
    校验危急值二级审核的合规性。
    1. 审核人必须具备指定角色资质
    2. 审核人不得与检测操作人为同一人
    3. 审核记录必须包含完整时间戳
    """
    # 获取审核人角色
    reviewer_role = await session.execute(
        select(UserRole.role_name).where(UserRole.user_id == reviewer_user_id)
    )
    role = reviewer_role.scalar_one_or_none()

    if role not in CRITICAL_VALUE_REVIEWER_ROLES:
        raise HTTPException(
            status_code=status.HTTP_403_FORBIDDEN,
            detail=f"危急值二级审核人员必须具备以下角色之一："
                   f"{CRITICAL_VALUE_REVIEWER_ROLES}，当前角色：{role}"
        )

    # 获取检测记录，校验审核人与操作人不为同一人
    exam = await session.get(Examination, examination_id)
    if exam.operator_id == reviewer_user_id:
        raise HTTPException(
            status_code=status.HTTP_400_BAD_REQUEST,
            detail="危急值二级审核人不得与检测操作人为同一人"
        )

    return True
