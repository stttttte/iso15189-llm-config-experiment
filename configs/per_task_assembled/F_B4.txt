# ISO 15189:2022 检验科文件体系生成指南

## 本套模板的用途

本套 MD 模板文件用于指导大语言模型 (LLM) 为医学实验室（检验科）生成符合 **ISO 15189:2022** 要求的质量管理体系文件。

**定位**：将"从零编写"变为"审核修改"，节省 70-80% 编写时间。

**输出物**：质量手册 + 程序文件(SOP) + 记录表单/作业指导书（三层文件结构）。

---

## 推荐使用的 LLM 模型

由于单个模板文件约 8K-12K tokens，请使用以下级别的模型：

| 推荐级别 | 模型 | 说明 |
|---------|------|------|
| 最佳 | Claude Opus / Claude Sonnet | 长上下文处理能力强，中文质量高 |
| 推荐 | GPT-4 / GPT-4o | 指令遵循度好 |
| 可用 | 其他 128K+ 上下文模型 | 需测试中文输出质量 |
| 不推荐 | GPT-3.5 / 4K 上下文模型 | 上下文不足，生成质量差 |

---

## 使用流程

```
步骤1：填写下方「检验科基本信息」
  ↓
步骤2：将本文件 + 01-质量手册-框架.md 一起发给 LLM
  ↓  LLM 生成质量手册初稿
步骤3：逐个发送 02~08 程序文件模板给 LLM
  ↓  每次附上本文件的「基本信息」部分作为上下文
  ↓  LLM 逐模块生成程序文件
步骤4：发送 09-记录表单清单.md 给 LLM
  ↓  LLM 生成配套表单
步骤5：发送 10-POCT专项.md（如有 POCT）
  ↓
步骤6：人工审核 → 签批发布 → 实施运行
```

**重要提示**：
- 每次发给 LLM 时，请把下方「基本信息」一并粘贴，确保 LLM 知道你的实验室具体情况
- 生成结果中所有 `【】` 包裹的内容为必须替换的占位符

## 标准原文要求（生成参考，不纳入正文）

> **§5.1 法人实体**：实验室或其所属组织应是能够承担法律责任的实体。
>
> **§5.2 实验室主任**：实验室应有一名实验室主任，对实验室承担总体责任。实验室主任的能力应适合其所承担的职责。实验室主任应具有相应资质、培训和/或经验。
>
> **§5.4 结构和权力**：
> - a) 实验室应有组织架构图，说明其在所属组织中的位置；
> - b) 明确管理层、技术运作和支持服务之间的关系；
> - c) 规定实验室各岗位的职责、权限和相互关系；
> - d) 确保沟通的充分性。

## 1. 目的

建立和维护实验室组织管理体系，明确法律实体地位、实验室主任资质与职责、组织架构与岗位权限、职责委托管理及沟通管理机制，确保实验室在清晰的组织框架下运行，满足 ISO 15189:2022 第 5.1、5.2、5.4 条款要求。

## 2. 适用范围

适用于【医院名称】医学检验科（以下简称"本实验室"）的组织管理活动，包括：

- 法律实体与资质证书管理
- 实验室主任任命、资质确认与职责分配
- 组织架构建立与维护
- 岗位职责与权限分配
- 职责委托管理
- 内外部沟通管理

### 6.1 法律实体与资质证书管理

#### 6.1.1 法律实体确认

本实验室隶属于【医院名称】（统一社会信用代码：【】），【医院名称】为依法设立的医疗机构法人实体，能够承担法律责任。

本实验室作为【医院名称】的内设科室（部门），其活动由所属法人实体承担法律责任。

#### 6.1.2 资质证书管理

本实验室应确保以下资质证书的持续有效：

| 序号 | 证书名称 | 发证机关 | 有效期 | 管理责任人 |
|------|---------|---------|--------|-----------|
| 4 | 【其他资质证书】 | 【】 | 【】 | 【】 |

### 6.3 职责委托管理

#### 6.3.1 委托原则

- 实验室主任可将部分职责委托给具备相应能力的人员
- 职责委托不免除实验室主任的总体责任
- 委托应正式记录，被委托人应签字确认
- 委托前应评估被委托人的能力和资质

#### 6.3.2 委托流程

```
实验室主任识别需委托的职责
        ↓
评估被委托候选人的能力和资质
        ↓
填写《职责委托记录表》(JYK-FM-05-001)
        ↓
实验室主任与被委托人双方签字确认
        ↓
通知相关人员
        ↓
定期评估委托执行效果（至少每年一次）
```

#### 6.3.3 委托记录内容

## 标准原文要求（生成参考，不纳入正文）

> **§5.3 实验室活动范围**：
> - a) 实验室应确定其活动范围，包括检验前过程、检验过程、检验后过程和咨询活动；
> - b) 实验室活动范围应可供服务用户获取；
> - c) 在不同场所或通过移动设备提供服务时，应满足本文件的要求。

---

## 1. 目的

建立实验室活动范围的确定、发布和管理程序，规范咨询服务的提供方式，确保实验室在确定的范围内开展活动，并为临床提供专业的咨询服务，满足 ISO 15189:2022 第 5.3 条款要求。

## 2. 适用范围

适用于本实验室活动范围的界定、变更、发布，以及咨询服务和多场所活动的管理。

## 3. 引用标准

| 标准编号 | 标准名称 |
|---------|---------|
| ISO 15189:2022 | 医学实验室——质量和能力要求 |
| CNAS-CL02:2023 | 医学实验室质量和能力认可准则 |
| CNAS-RL01 | 实验室认可规则 |
| CNAS-RL02 | 能力验证规则 |

## 4. 术语和定义

| 术语 | 定义 |
|------|------|
| 实验室活动 | 实验室执行的与检验有关的所有活动，包括检验前、检验中、检验后过程及咨询活动 |
| 认可范围 | 经 CNAS 认可的实验室检验活动范围 |
| 咨询服务 | 实验室为临床诊疗提供的检验结果解读、检验项目选择建议等专业支持 |
| POCT | 即时检验（Point-of-Care Testing），在患者身边或附近进行的检验 |
| 多场所 | 在主实验室以外的固定或临时场所开展的检验活动 |

## 5. 职责

| 岗位/角色 | 职责 |
|----------|------|
| 实验室主任 | 批准活动范围清单和变更，主导咨询服务规划 |
| 技术负责人 | 组织活动范围的评审和更新，指导专业咨询 |
| 质量负责人 | 维护活动范围清单文件，管理认可范围声明 |
| 各专业组组长 | 提交本组活动范围清单和变更申请，实施咨询服务 |

## 6. 程序内容

### 6.1 活动范围清单

#### 6.1.1 清单内容要求

活动范围清单应按专业组分类，覆盖检验前、检验中、检验后全过程及咨询活动。清单应至少包含以下信息：

| 栏目 | 说明 |
|------|------|
| 专业组 | 如临床化学、临床免疫、临床血液等 |
| 检验项目名称 | 如丙氨酸氨基转移酶（ALT） |
| 检验方法 | 如速率法（IFCC参考方法）|
| 主要仪器 | 型号和编号 |
| 样本类型 | 如血清、EDTA全血等 |
| 报告时间（TAT）| 常规/急诊 |
| 检验阶段 | 检验前/检验中/检验后/咨询 |
| 认可状态 | 已认可/待认可/非认可 |

#### 6.1.2 活动范围清单示例

> ⚠️ 以下为示例，请替换为实际检验项目

**临床化学组：**

| 检验项目 | 方法 | 仪器 | 样本类型 | TAT（常规/急诊） | 认可状态 |
|---------|------|------|---------|----------------|---------|

**检验前活动：**

| 活动内容 | 适用范围 | 责任岗位 |
|---------|---------|---------|
| 采样指南编制与发布 | 全科室 | 技术负责人 |
| 标本接收和验收 | 全科室 | 标本接收岗 |
| 标本运输条件管理 | 全科室 | 各专业组 |

**检验后活动：**

| 活动内容 | 适用范围 | 责任岗位 |
|---------|---------|---------|
| 结果审核与发布 | 全科室 | 授权审核人员 |
| 标本保存与处置 | 全科室 | 各专业组 |
| 咨询服务 | 全科室 | 见 6.4 |

#### 6.1.3 清单更新与变更

活动范围发生以下变化时，应及时更新清单：
- 新增检验项目
- 停止检验项目
- 变更检验方法
- 变更主要检验设备
- 新增/关闭检验场所

变更流程：
```
专业组提出变更申请 → 技术负责人评审 → 实验室主任批准
        → 更新活动范围清单 → 通知相关科室和服务用户
```

### 6.2 认可范围声明管理

#### 6.2.1 认可范围声明

如本实验室已获得 CNAS 认可，应：
- 在官方文件中正确使用 CNAS 认可标识
- 明确区分认可范围内和认可范围外的检验项目
- 在检验报告上正确标识认可状态
- 认可范围发生变更时，按 CNAS 规则办理扩项/缩项手续

#### 6.2.2 非认可范围项目

对于超出认可范围的检验项目：
- 应在报告中明确标识为非认可范围项目
- 同样应满足质量管理体系要求
- 有计划地纳入认可申请范围（如适用）

### 6.3 多场所活动管理

#### 6.3.1 场所清单

| 场所名称 | 类型 | 地址 | 开展项目 | 管理责任人 |
|---------|------|------|---------|-----------|
| 主实验室 | 固定 | 【医院地址 + 楼层】 | 全部检验项目 | 实验室主任 |
| 【其他场所】 | 【】 | 【】 | 【】 | 【】 |

#### 6.3.2 多场所管理要求

- 所有场所的活动均纳入统一的质量管理体系
- POCT 设备由检验科统一管理或参与管理
- POCT 操作人员须经本实验室培训并考核合格
- POCT 质控纳入本实验室质量控制体系

### 6.4 咨询服务管理

#### 6.4.1 咨询服务范围

| 服务类型 | 内容 | 服务方式 | 责任人 |
|---------|------|---------|--------|
| 检验项目选择咨询 | 为临床推荐合适的检验项目组合 | 电话/会诊 | 实验室主任/技术负责人 |
| 检验结果解读 | 解释异常结果的临床意义 | 电话/面谈/书面 | 当值高级专业人员 |
| 检验前咨询 | 采样要求、患者准备、标本运输 | 电话/采样指南 | 各专业组 |
| 统计分析服务 | 为临床科研提供检验数据统计分析 | 书面报告 | 技术负责人 |
| 新项目推荐 | 主动向临床推荐新检验技术/项目 | 通知/培训 | 实验室主任 |

#### 6.4.2 主动咨询与被动咨询

**主动咨询**（实验室主动发起）：
- 危急值通报及处置建议
- 异常结果组合的提示（如血液学参数提示 DIC 等）
- 新项目/新方法的临床推荐
- 检验相关临床路径建议

**被动咨询**（临床发起请求）：
- 临床来电/来函咨询
- 会诊请求
- 检验报告疑问解答

#### 6.4.3 咨询记录

所有正式咨询（排除简单的日常沟通）应记录在《咨询服务记录表》(JYK-FM-05-008)，内容包括：
- 咨询日期和时间
- 咨询方（科室、姓名、联系方式）
- 咨询问题描述
- 答复内容和答复人
- 需进一步跟踪的事项

## 1. 目的

建立质量方针和质量目标的制定、发布、宣贯、监测和评审程序，确保质量方针为管理体系提供方向，质量目标可量化、可跟踪、可改进，满足 ISO 15189:2022 第 5.5 条款要求。

## 2. 适用范围

适用于本实验室质量方针的制定与传达、质量目标的设定与监测、目标达成情况的评审与改进。

## 3. 引用标准

| 标准编号 | 标准名称 |
|---------|---------|
| ISO 15189:2022 | 医学实验室——质量和能力要求 |
| CNAS-CL02:2023 | 医学实验室质量和能力认可准则 |

## 4. 术语和定义

| 术语 | 定义 |
|------|------|
| 质量方针 | 由管理层正式发布的实验室在质量方面的总体方向和意图 |
| 质量目标 | 在质量方面所追求的目的，与质量方针一致，通常是可测量的 |
| SMART 原则 | 目标设定原则：具体(Specific)、可测量(Measurable)、可达到(Achievable)、相关性(Relevant)、有时限(Time-bound) |

## 5. 职责

| 岗位/角色 | 职责 |
|----------|------|
| 实验室主任 | 主持制定和批准质量方针及目标，主持年度评审 |
| 质量负责人 | 组织质量方针和目标的编制、宣贯和监测，汇总分析数据 |
| 技术负责人 | 参与技术相关目标的制定和评审，提供技术数据支持 |
| 各专业组组长 | 负责本组目标数据的收集、统计和上报 |
| 全体人员 | 理解质量方针，为实现质量目标做出贡献 |

## 6. 程序内容

### 6.1 质量方针管理

#### 6.1.1 质量方针的制定

质量方针的制定应遵循以下原则：
- 与本实验室的宗旨和服务范围相适应
- 包含对满足要求和持续改进的承诺
- 提供制定和评审质量目标的框架
- 语言简明，易于全体人员理解

> ⚠️ 以下为质量方针示例，请替换为实际方针

```
质量方针（示例）：

"科学公正、准确及时、持续改进、满足需求。

本实验室致力于以专业的技术能力和严谨的质量管理，为临床诊疗
提供科学、准确、及时的检验服务。我们承诺：

- 遵守国家法律法规和行业标准，坚持科学公正；
- 持续改进检验全过程质量，不断提升服务水平；
- 满足患者和临床科室的合理需求，为医疗安全保驾护航。"
```

#### 6.1.2 质量方针的发布与展示

| 发布/展示方式 | 具体措施 | 责任人 |
|-------------|---------|--------|
| 文件发布 | 纳入质量手册正文 | 质量负责人 |
| 员工手册 | 发放给每位新入职人员 | 质量负责人 |

#### 6.1.3 质量方针的宣贯

- 新员工入职培训时进行质量方针宣贯
- 每年至少组织 1 次全员质量方针学习
- 宣贯后进行理解程度考核（可纳入年度培训考核）
- 保留宣贯和考核记录

#### 6.1.4 质量方针的评审

- 质量方针应在每次管理评审时进行适宜性评审
- 出现以下情况时应评审是否需要修订：
  - 实验室活动范围发生重大变化
  - 法律法规或标准要求发生重大变化
  - 组织结构发生重大变化
- 修订后按文件控制程序重新发布

### 6.2 质量目标管理

#### 6.2.1 质量目标的设定

质量目标应符合 SMART 原则，与质量方针一致。目标类别包括但不限于：

| 类别 | 说明 |
|------|------|
| 检验质量目标 | 室间质评合格率、室内质控合格率等 |
| 服务时效目标 | TAT达标率等 |
| 安全目标 | 生物安全事故发生率等 |
| 满意度目标 | 临床满意度等 |
| 能力建设目标 | 人员培训、新项目开展等 |
| 持续改进目标 | 不符合项整改率等 |

#### 6.2.2 质量目标清单

> ⚠️ 以下为示例目标清单，请根据实际情况替换

| 序号 | 目标名称 | 目标值 | 监测周期 | 数据来源 | 责任人 |
|------|---------|--------|---------|---------|--------|

### 6.3 目标达标情况跟踪与趋势分析

#### 6.3.1 数据收集与统计

- 各责任人按监测周期收集目标数据
- 使用《质量目标监测统计表》(JYK-FM-05-013) 记录
- 质量负责人汇总各项目标数据

#### 6.3.2 趋势分析

- 每季度对质量目标完成情况进行趋势分析
- 分析方法：与目标值比较、与上一周期比较、趋势图（折线图/柱状图）
- 识别持续恶化趋势并提前预警
- 分析结果汇总至管理评审输入

#### 6.3.3 目标达成报告

- 每半年编制《质量目标达成情况报告》
- 报告内容：各目标完成数据、趋势分析、达标/未达标判定、原因分析
- 报告提交实验室主任审阅

### 6.4 未达标项改进措施

#### 6.4.1 未达标判定

当某项质量目标在一个监测周期内未达到目标值时，判定为未达标。

#### 6.4.2 改进流程

```
发现目标未达标
      ↓
责任人分析原因（使用鱼骨图/5Why 等方法）
      ↓
制定纠正措施和改进计划
      ↓
质量负责人审核改进计划
      ↓
实施改进措施
      ↓
跟踪验证改进效果
      ↓
连续 2 个监测周期达标 → 关闭
未达标 → 升级措施 → 管理评审讨论
```

#### 6.4.3 改进记录

未达标改进应记录在《质量目标未达标改进跟踪表》(JYK-FM-05-014)，内容包括：
- 目标名称及未达标数据
- 原因分析
- 纠正措施和时限
- 责任人
- 验证结果和关闭日期

### 6.5 目标年度评审和修订

#### 6.5.1 年度评审

- 每年 **12 月**（或结合管理评审时间）对质量目标进行年度评审
- 评审内容：
  - 各项目标的年度达成情况
  - 目标值是否仍然适宜（是否需要提高/调整）
  - 是否需要增加/删除目标项
  - 与质量方针的一致性

#### 6.5.2 目标修订原则

- 已持续达标的目标，考虑适当提高目标值
- 始终远超目标值的指标，应提高目标值以体现持续改进
- 因客观条件变化导致不适用的目标，可删除并说明原因
- 新增目标应有充分的数据基线

#### 6.5.3 修订审批

目标修订应经实验室主任批准，并在下一年度开始前发布新版质量目标清单。

## 标准原文要求（生成参考，不纳入正文）

> **§8.1 总则**：
> - 实验室应建立、文件化、实施和维护管理体系，能够支持和证明一致地满足本文件要求。
> - 管理体系至少应包括：a) 质量方针和目标；b) 活动范围；c) 能力和素质要求的信息（第6章）的实施证据；d) 对承诺和义务的证据（第7章）；e) 程序和记录的证明文件。
>
> **§8.2 管理体系文件**：
> - a) 管理层应确保将方针、程序和活动传达给适当的人员，并使其理解和实施；
> - b) 所有人员应能获取适用于其职责的管理体系文件；
> - c) 管理层应确保人员意识到管理体系的有效性、自身活动的重要性，以及对管理体系改进的贡献。

## 1. 目的

建立管理体系文件的整体框架和管理要求，明确管理体系的构成、文件层级、人员意识培训和文件访问权限管理，确保管理体系得到有效建立、实施和维护，满足 ISO 15189:2022 第 8.1、8.2 条款要求。

## 2. 适用范围

适用于本实验室管理体系文件的框架设计、层级划分、意识培训和访问权限管理。

> **注**：文件的具体编写、审批、发布、修订、作废等操作流程见 JYK-SOP-05-005《文件控制程序》；记录的创建、修改、保留等见 JYK-SOP-05-006《记录控制程序》。

## 6. 程序内容

### 6.1 管理体系框架

#### 6.1.1 管理体系范围

本实验室管理体系覆盖：
- 检验前过程（标本采集、运输、接收、处理）
- 检验过程（方法确认/验证、质量控制、结果报告）
- 检验后过程（结果发布、标本保存/处置、咨询服务）
- 支持过程（人员、设备、试剂、环境设施、信息管理）
- 管理过程（组织与治理、风险管理、持续改进）
- 适用于所有检验场所（主实验室和 POCT/采样点）

#### 6.1.2 管理体系依据

本管理体系依据以下标准建立：
- ISO 15189:2022《医学实验室——质量和能力要求》
- CNAS-CL02:2023《医学实验室质量和能力认可准则》
- 相关国家法律法规和行业标准

### 6.2 文件层级结构

本实验室管理体系文件采用 **四层结构**：


### 6.3 管理体系文件构成

#### 6.3.1 按标准章节的文件构成

依据 ISO 15189:2022 第 8.1 条款，管理体系文件应至少包含以下内容：

| 序号 | 文件类别 | 对应标准要求 | 文件示例 |
|------|---------|-------------|---------|
| 1 | 质量方针和目标文件 | §8.1 a) + §5.5 | 质量手册第5章、质量目标清单 |
| 2 | 活动范围文件 | §8.1 b) + §5.3 | 活动范围清单、认可范围声明 |
| 3 | 能力和素质的实施证据 | §8.1 c)（对应第6章） | 人员档案、培训记录、能力评估记录、设备档案、校准记录、试剂验收记录等 |
| 4 | 承诺和义务的证据 | §8.1 d)（对应第7章） | 公正性声明、保密协议、安全记录、合同评审记录等 |
| 5 | 程序和记录的证明文件 | §8.1 e) | 全套程序文件、作业指导书和记录表单 |

#### 6.3.2 按管理过程的文件构成

| 管理过程 | 核心文件 |
|---------|---------|
| 组织与治理（第5章） | 组织管理程序、活动范围程序、质量方针目标管理程序 |
| 资源管理（第6章） | 人员管理程序、设备管理程序、试剂管理程序、环境设施管理程序、信息管理程序 |
| 过程要求（第7章） | 检验前/中/后过程程序、公正性程序、保密性程序 |
| 管理体系要求（第8章） | 文件控制程序、记录控制程序、风险管理程序、改进程序、不符合管理程序、内部审核程序、管理评审程序 |

### 6.4 管理体系意识培训

#### 6.4.1 培训目标

确保全体人员理解：
- 管理体系的目的和意义
- 质量方针和质量目标
- 自身活动对管理体系有效性的贡献
- 不符合管理体系要求的后果
- 持续改进的重要性

#### 6.4.2 培训安排

| 培训类型 | 对象 | 频次 | 内容 |
|---------|------|------|------|
| 新员工入职培训 | 新入职人员 | 入职时 | 管理体系概述、质量方针、岗位职责、相关文件 |
| 标准培训 | 相关管理人员 | 标准更新时 | ISO 15189 标准条款解读和变更要点 |
| 文件宣贯培训 | 相关人员 | 文件更新时 | 新发布或修订文件的内容和要求 |

#### 6.4.3 意识评估

- 通过培训考核、内审访谈、日常检查等方式评估人员对管理体系的理解程度
- 发现意识不足时，安排针对性补训
- 意识评估结果纳入人员能力评估

### 6.5 文件访问权限管理

#### 6.5.1 权限分级

| 权限级别 | 适用人员 | 可访问文件范围 |
|---------|---------|--------------|
| 全部访问 | 实验室主任、质量负责人、技术负责人 | 全部管理体系文件 |
| 部门级 | 各专业组组长 | 全部通用文件 + 本组专业文件 |
| 岗位级 | 普通技术人员 | 通用文件 + 与岗位职责相关的专业文件 |
| 受限访问 | 实习生/进修人员 | 经批准的部分文件 |

#### 6.5.2 文件获取方式

| 获取方式 | 说明 |
|---------|------|
| 纸质文件 | 受控纸质版存放于各专业组指定位置，由文件管理员管理 |
| 现场张贴 | 关键操作流程、安全须知等张贴于工作现场 |

#### 6.5.3 权限变更

- 人员岗位变动时，及时调整文件访问权限
- 人员离职时，收回文件访问权限并回收纸质受控文件
- 权限变更记录应留存

#### 6.2 文件全生命周期管理

##### 6.2.1 文件创建

文件创建要求：
- 按本实验室统一的文件格式模板编写
- 内容应准确、完整、可操作
- 引用条款应标注准确
- 编制人负责初稿编写

##### 6.2.2 文件审批

文件审批权限：

| 文件类型 | 编制 | 审核 | 批准 |
|---------|------|------|------|
| 质量手册 | 质量负责人 | 技术负责人 | 实验室主任 |
| 管理类程序文件 | 质量负责人/指定人员 | 技术负责人 | 实验室主任 |
| 技术类程序文件 | 专业组组长/指定人员 | 技术负责人 | 实验室主任 |
| 作业指导书 | 操作人员/组长 | 专业组组长 | 技术负责人 |
| 记录表单模板 | 使用人员 | 质量负责人 | 技术负责人 |

审批流程：
```
编制人编写初稿
      ↓
送审核人审核（内容准确性、格式规范性、标准符合性）
      ↓
审核通过 → 送批准人批准
审核不通过 → 退回编制人修改
      ↓
批准人签字 → 文件管理员登记编号
```

##### 6.2.3 文件发布与分发

- 批准后由文件管理员统一发布
- 受控文件加盖 **红色"受控"章**，标注分发编号
- 电子版文件设置为只读，存放在受控文件夹
- 分发时收件人在《文件分发/回收记录表》(JYK-FM-05-021) 上签收
- 新文件发布时，相关人员须确认接收

##### 6.2.4 文件修订

文件修订的触发条件：
- 法律法规、标准要求发生变化
- 管理体系运行中发现问题
- 内部审核/管理评审提出修改需求
- 检验方法或流程发生变更
- 定期评审发现不适用

修订流程：
```
识别修订需求 → 填写《文件修订申请表》(JYK-FM-05-022)
      ↓
质量负责人评审修订必要性
      ↓
批准修订 → 编制人修订文件
      ↓
按原审批权限重新审批
      ↓
文件管理员更新版本号和修订记录
      ↓
发布新版 → 回收旧版
```

修订标注要求：
- 修订内容在文件"修订记录"表中注明
- 重大修订应组织相关人员培训宣贯

##### 6.2.5 文件作废

- 被新版本替代的旧版文件应及时作废
- 作废纸质文件由文件管理员统一回收，加盖 **"作废"章**
- 留存 1 份作废文件作为历史记录，其余销毁
- 电子版作废文件移至"作废文件"文件夹，设置不可编辑
- 更新受控文件清单，标注作废信息

#### 6.3 版本管理和历史追溯

- 所有文件版本变更均记录在文件的"修订记录"表中
- 文件管理员维护《文件版本变更历史台账》(JYK-FM-05-023)
- 至少保留每份文件最近 3 个历史版本
- 需要时可追溯到任意历史版本

#### 6.4 受控文件清单维护

文件管理员应维护《受控文件清单》(JYK-FM-05-020)，包含以下信息：

| 栏目 | 说明 |
|------|------|
| 文件编号 | 唯一编号 |
| 文件名称 | 完整名称 |
| 版本号 | 当前有效版本 |

### 6. 程序内容

#### 6.1 记录的创建

##### 6.1.1 记录创建基本要求

所有记录应满足以下要求：

| 要求 | 说明 |
|------|------|
| 及时性 | 在活动发生时或完成后立即记录，不得事后补记（特殊情况除外） |
| 准确性 | 客观、真实地记录实际情况，不得篡改或伪造 |
| 完整性 | 按表单要求填写所有栏目，不得随意空置（无内容填"/" 或"无"） |
| 清晰性 | 字迹工整、清晰可辨，使用永久性书写工具（黑色签字笔） |
| 可追溯性 | 记录人应签名并标注日期 |

##### 6.1.2 纸质记录要求

- 使用本实验室统一的记录表单（受控模板）
- 禁止使用铅笔、修正液、修正带
- 记录内容不得涂改至无法辨认原始内容

##### 6.1.3 电子记录要求

- LIS 系统和仪器自动生成的记录应具有审计追踪功能
- 电子记录应有操作者身份验证（用户名/密码）
- 电子记录的备份策略见信息管理相关程序
- 电子签名应与纸质签名具有同等效力（如适用）

#### 6.2 记录的修改（留痕可追溯）

##### 6.2.1 纸质记录修改

修改规则：
```
                    错误内容
                    ────────
正确写法：  在错误内容上划一条横线（仍可辨认原始内容）
            旁边写上正确内容
            签名 + 日期

示例：
     血红蛋白：120 135 g/L
                ───
                     张三 2026-01-15

禁止：使用修正液/修正带覆盖
禁止：涂改至无法辨认原文
禁止：撕毁或更换记录页
```

##### 6.2.2 电子记录修改

- 修改应通过系统审计追踪功能自动记录
- 审计追踪应记录：修改人、修改时间、修改前内容、修改后内容、修改原因
- 禁止通过数据库直接修改绕过审计追踪
- LIS 系统应对关键数据的修改设置权限控制

##### 6.2.3 重大修改审批

涉及以下情况的记录修改应经上级审批：
- 已发布检验报告的数据修改
- 质控数据的修改
- 校准数据的修改
- 其他可能影响检验结果准确性的修改

#### 6.3 记录保留期限配置

##### 6.3.1 保留期限标准

| 记录类别 | 保留期限 | 说明 |
|---------|---------|------|

> **注**：如法规要求的保留期限长于上述规定，应从其规定。

##### 6.3.2 保留期限评审

- 记录保留期限应定期评审（至少每 2 年一次）
- 评审时考虑法规要求变化、认可要求变化、医疗诉讼时效等因素
- 保留期限变更需经实验室主任批准

#### 6.4 记录归档和索引管理

##### 6.4.1 归档流程

```
记录使用完毕（当月/当季/当年度结束）
      ↓
专业组整理、自查完整性
      ↓
按归档要求装订整理
      ↓
移交记录管理员归档
      ↓
记录管理员验收、编制索引
      ↓
入库保管
```

##### 6.4.2 归档要求

| 要求项 | 具体内容 |
|--------|---------|
| 分类方式 | 按专业组 + 记录类型 + 年度分类 |
| 装订顺序 | 按时间顺序排列，最新在后 |
| 标识要求 | 档案盒/文件夹应标注：专业组、记录类型、时间范围、档案编号 |
| 索引登记 | 在《记录归档索引台账》(JYK-FM-05-030) 中登记 |

##### 6.4.3 存储条件

| 记录类型 | 存储位置 | 环境要求 |
|---------|---------|---------|

##### 6.4.4 记录检索

- 记录应可在合理时间内检索到（目标：纸质记录 **30 分钟内**，电子记录 **即时**）
- 通过索引台账或电子检索系统查找
- 借阅纸质记录应在《记录借阅登记表》(JYK-FM-05-031) 上登记，借阅后归还

#### 6.5 记录处置

##### 6.5.1 到期处置

记录超过保留期限后：
```
记录管理员编制到期记录清单
      ↓
质量负责人审核
      ↓
实验室主任批准销毁
      ↓
执行销毁（纸质：碎纸机；电子：安全删除）
      ↓
填写《记录销毁记录表》(JYK-FM-05-032)
      ↓
销毁需两人在场，共同签字确认
```

##### 6.5.2 特殊情况

- 涉及医疗纠纷/诉讼的记录，不论是否到期，均不得销毁，直至纠纷结案
- 认可评审期间，相关记录不得销毁
- 确需延长保存的记录，经实验室主任批准后延期
