# ISO 15189:2022 检验科文件体系生成指南

## 本套模板的用途

本套 MD 模板文件用于指导大语言模型 (LLM) 为医学实验室（检验科）生成符合 **ISO 15189:2022** 要求的质量管理体系文件。

**定位**：将"从零编写"变为"审核修改"，节省 70-80% 编写时间。

**输出物**：质量手册 + 程序文件(SOP) + 记录表单/作业指导书（三层文件结构）。

---

## 推荐使用的 LLM 模型

由于单个模板文件约 8K-12K tokens，请使用以下级别的模型：

| 推荐级别 | 模型 | 说明 |
|---------|------|------|
| 最佳 | Claude Opus / Claude Sonnet | 长上下文处理能力强，中文质量高 |
| 推荐 | GPT-4 / GPT-4o | 指令遵循度好 |
| 可用 | 其他 128K+ 上下文模型 | 需测试中文输出质量 |
| 不推荐 | GPT-3.5 / 4K 上下文模型 | 上下文不足，生成质量差 |

## 使用流程

```
步骤1：填写下方「检验科基本信息」
  ↓
步骤2：将本文件 + 01-质量手册-框架.md 一起发给 LLM
  ↓  LLM 生成质量手册初稿
步骤3：逐个发送 02~08 程序文件模板给 LLM
  ↓  每次附上本文件的「基本信息」部分作为上下文
  ↓  LLM 逐模块生成程序文件
步骤4：发送 09-记录表单清单.md 给 LLM
  ↓  LLM 生成配套表单
步骤5：发送 10-POCT专项.md（如有 POCT）
  ↓
步骤6：人工审核 → 签批发布 → 实施运行
```

**重要提示**：
- 每次发给 LLM 时，请把下方「基本信息」一并粘贴，确保 LLM 知道你的实验室具体情况
- 生成结果中所有 `【】` 包裹的内容为必须替换的占位符
- 所有标注 `⚠️ 示例，请替换` 的文本为示例内容，必须替换为真实信息

## SOP 1: JYK-SOP-08-001《风险和改进机会管理程序》

**对应标准条款**：ISO 15189:2022 §5.6 风险管理 + §8.5 应对风险和改进机会的行动

### 1. 目的

建立风险和改进机会的识别、评估、应对和监控程序，系统性地管理实验室活动中的风险，并持续寻求改善患者护理的机会。

### 2. 适用范围

适用于本实验室检验前、检验中、检验后全过程，以及管理活动和支持过程中的风险管理。

### 3. 引用标准

- ISO 15189:2022 §5.6, §8.5
- ISO 22367 医学实验室风险管理
- ISO 35001 实验室生物风险管理

### 4. 术语和定义

- **风险**：不确定性对目标的影响（包括正面和负面影响）
- **风险评估**：风险识别、风险分析和风险评价的全过程
- **改进机会**：可以提高管理体系有效性、改善患者护理的可行措施

### 5. 职责

| 角色 | 职责 |
|------|------|
| 实验室主任 | 批准风险管理方案，评估风险管理过程有效性 |
| 质量负责人 | 组织风险识别和评估，跟踪风险应对措施的实施 |
| 各组组长 | 识别本专业组内的风险，执行风险应对措施 |
| 全体人员 | 报告发现的风险和改进机会 |

### 6. 程序内容

> **LLM 生成指令**：按以下框架展开，每一步给出具体操作说明。

#### 6.1 风险识别 [§5.6(a), §8.5.1]

**标准要求**：
> 实验室管理层应建立、实施和维护相关流程，以识别对患者造成伤害的风险以及与其检查和活动相关的改善患者护理的机会。

**程序内容**：

6.1.1 风险识别的范围应覆盖：
- 检验前过程风险（样本采集、运输、接收、前处理）
- 检验过程风险（方法选择、设备故障、试剂失效、质控失控）
- 检验后过程风险（结果审核、报告发布、危急值通知）
- 支持过程风险（人员能力、设施环境、信息系统、供应商）
- 管理过程风险（公正性、保密性、文件控制）

6.1.2 风险识别方法：
- 定期风险评估会议（至少每年一次）
- 日常工作中的风险报告
- 不合格工作分析
- 投诉和反馈分析
- 内部审计发现
- EQA/IQC 失控事件分析
- 法规和标准变化评估

6.1.3 风险识别工具：

⚠️ 示例，请替换：

| 过程 | 潜在风险 | 可能后果 |
|------|---------|---------|
| 样本采集 | 患者身份识别错误 | 结果张冠李戴，影响诊疗 |
| 样本运输 | 温度超出允许范围 | 分析物降解，结果不准确 |
| 检验过程 | IQC失控未发现 | 发布错误结果 |
| 结果报告 | 危急值未及时通知 | 延误患者治疗 |
| 设备管理 | 维护不及时 | 设备故障导致停检 |

#### 6.2 风险分析与评价

6.2.1 风险等级评定矩阵：

| | 后果-轻微(1) | 后果-中等(2) | 后果-严重(3) | 后果-灾难性(4) |
|---|---|---|---|---|
| **可能性-极少(1)** | 1-低 | 2-低 | 3-中 | 4-中 |
| **可能性-偶尔(2)** | 2-低 | 4-中 | 6-高 | 8-高 |
| **可能性-经常(3)** | 3-中 | 6-高 | 9-极高 | 12-极高 |
| **可能性-频繁(4)** | 4-中 | 8-高 | 12-极高 | 16-极高 |

6.2.2 风险分级处理：
- **极高(9-16)**：立即采取措施，报告主任
- **高(6-8)**：制定纠正措施计划，限期整改
- **中(3-4)**：纳入改进计划，定期监控
- **低(1-2)**：记录备案，常规监控

#### 6.3 风险应对 [§5.6(a), §8.5.2]

**标准要求**：
> 制定措施应对风险和改善机会。

6.3.1 风险应对策略：
- **消除**：去除风险源
- **降低**：降低风险发生的可能性或后果严重程度
- **转移**：与其他方共担风险
- **接受**：在可接受范围内保留风险

6.3.2 改进机会管理：
- 识别改善患者护理的机会
- 评估改进的可行性和预期效果
- 制定改进行动计划
- 实施、监控和评价改进效果

#### 6.4 风险监控和评审 [§5.6(b)]

**标准要求**：
> 实验室主任应确保对这些过程的有效性进行评估，并在发现无效时进行修改。

6.4.1 风险登记册应定期更新（至少每年一次）
6.4.2 风险应对措施的有效性应定期评审
6.4.3 新的风险应及时补充到风险登记册中
6.4.4 风险管理情况应作为管理评审的输入

### 7. 相关文件

- JYK-SOP-08-002《持续改进管理程序》

## SOP 2: JYK-SOP-08-002《持续改进管理程序》

**对应标准条款**：ISO 15189:2022 §8.6

### 1. 目的

建立持续改进的机制，通过系统性收集和分析反馈信息，识别改进机会，实施改进措施并评价其效果。

### 2. 适用范围

适用于本实验室管理体系和检验活动的全过程改进。

### 3. 引用标准

- ISO 15189:2022 §8.6

### 4. 职责

| 角色 | 职责 |
|------|------|
| 实验室主任 | 推动持续改进文化，批准重大改进项目 |
| 质量负责人 | 汇总改进信息来源，组织改进项目实施 |
| 各组组长 | 在本组内推进改进措施 |
| 全体人员 | 提交改进建议 |

### 5. 程序内容

#### 5.1 改进信息来源 [§8.6.1]

**标准要求**：
> 实验室应持续改进管理体系的适宜性、充分性和有效性。

改进信息来源包括：
- 质量指标监控数据（§8.8.2）
- 内部审计结果（§8.8.3）
- 管理评审输出（§8.9）
- 不符合项和纠正措施分析（§8.7）
- 风险评估结果（§8.5）
- EQA 结果分析
- IQC 趋势分析
- 患者和用户反馈
- 投诉分析
- 员工建议
- 应急演练发现的问题

#### 5.2 反馈管理 [§8.6.2]

**标准要求**：
> 实验室患者、用户和人员反馈。

5.2.1 患者满意度调查（至少半年一次）
5.2.2 临床科室满意度调查（至少每年一次）
5.2.3 员工满意度调查和建议收集机制
5.2.4 反馈信息的汇总分析和改进措施

#### 5.3 改进项目管理

5.3.1 改进项目立项（项目名称、负责人、预期目标、完成期限）
5.3.2 改进措施实施
5.3.3 改进效果评价
5.3.4 改进成果固化（纳入程序文件或操作规范）

## SOP 3: JYK-SOP-08-003《不符合项和纠正措施管理程序》

**对应标准条款**：ISO 15189:2022 §8.7

### 1. 目的

建立不符合项的识别、记录、处理和纠正措施的实施程序，防止不符合项再次发生。

### 2. 适用范围

适用于管理体系运行和检验活动中发现的所有不符合项。

### 3. 引用标准

- ISO 15189:2022 §8.7
- JYK-SOP-07-006《不合格工作管理程序》（§7.5 的技术层面不合格）

### 4. 术语和定义

- **不符合项**：未满足规定要求
- **纠正**：为消除已发现的不符合所采取的措施
- **纠正措施**：为消除不符合的原因并防止再发生所采取的措施

### 5. 职责

| 角色 | 职责 |
|------|------|
| 质量负责人 | 统一管理不符合项，组织根因分析，跟踪纠正措施 |
| 各部门/组 | 报告不符合项，执行纠正措施 |
| 实验室主任 | 审批重大不符合项的纠正措施计划 |

### 6. 程序内容

#### 6.1 不合格发生时的行动 [§8.7.1]

**标准要求**：
> 当实验室活动或检查结果的任何方面不符合其自身的程序、质量规范或用户要求时，实验室应有一个过程。

6.1.1 不符合项来源：
- 内部审计发现
- 管理评审发现
- 日常质量监督发现
- IQC/EQA 失控
- 投诉处理过程
- 设备/试剂不良事件
- 外部评审（CNAS评审、卫生行政部门检查）
- 员工报告
- 患者安全事件

6.1.2 不符合项登记：
- 发现人填写不符合项报告（描述事实、发现时间、涉及范围）
- 质量负责人确认和分类（一般/严重/关键）

6.1.3 即时措施：
- 评估对患者结果的影响
- 必要时停止检查、扣留报告
- 通知相关方（临床科室、患者）
- 追溯已发布的可能受影响的结果

#### 6.2 纠正措施 [§8.7.2]

6.2.1 根因分析（使用 5-Why、鱼骨图等工具）
6.2.2 制定纠正措施计划（措施内容、责任人、完成期限）
6.2.3 实施纠正措施
6.2.4 验证纠正措施的有效性
6.2.5 必要时修订相关程序文件

#### 6.3 不符合项和纠正措施记录 [§8.7.3]

6.3.1 保留完整记录：不符合描述、根因分析、纠正措施、有效性验证
6.3.2 定期汇总分析不符合项趋势
6.3.3 作为管理评审的输入

## SOP 4: JYK-SOP-08-004《内部审计管理程序》

**对应标准条款**：ISO 15189:2022 §8.8

### 1. 目的

建立内部审计程序，定期评价管理体系的符合性和有效性，识别改进机会。

### 2. 适用范围

适用于管理体系覆盖的所有过程和活动。

### 3. 引用标准

- ISO 15189:2022 §8.8
- ISO 19011 管理体系审核指南

### 4. 职责

| 角色 | 职责 |
|------|------|
| 实验室主任 | 批准年度内审计划，评审内审报告 |
| 质量负责人 | 制定内审计划，组建审核组，编制审核报告 |
| 内审员 | 执行审核活动，编写审核发现 |
| 被审核部门 | 配合审核，实施纠正措施 |

### 5. 程序内容

#### 5.1 内审策划 [§8.8.3]

**标准要求**：
> 实验室应按策划的时间间隔进行内部审计。

5.1.1 年度内审计划编制（覆盖管理体系所有要素，12个月内至少审核一次）
5.1.2 审核频率：根据过程的重要性和以前审核结果确定
5.1.3 审核范围：依据 ISO 15189:2022 条款 + CNAS 应用说明

#### 5.2 审核准备

5.2.1 组建审核组（审核员应独立于被审核活动）
5.2.2 编制审核检查表（按条款逐条准备）
5.2.3 审核通知（提前通知被审核部门）

#### 5.3 审核实施

5.3.1 首次会议
5.3.2 现场审核（查阅文件和记录、观察操作、询问人员）
5.3.3 审核发现分类：符合 / 不符合（一般/严重）/ 观察项
5.3.4 末次会议（通报审核发现）

#### 5.4 审核报告

5.4.1 编制内审报告（审核范围、依据、发现汇总、结论）
5.4.2 报告提交主任审批

#### 5.5 不符合整改跟踪

5.5.1 被审核方制定纠正措施计划
5.5.2 验证纠正措施的实施和有效性
5.5.3 关闭不符合项

#### 5.6 质量指标 [§8.8.2]

**标准要求**：

5.6.1 质量指标清单（⚠️ 示例，请替换）：

5.6.2 质量指标数据收集和统计分析
5.6.3 质量指标趋势分析和目标达标评价
5.6.4 质量指标作为管理评审输入

## SOP 5: JYK-SOP-08-007《管理评审程序》

**对应标准条款**：ISO 15189:2022 §8.9

### 1. 目的

建立管理评审程序，确保管理体系的持续适宜性、充分性和有效性。

### 2. 适用范围

适用于实验室管理层对管理体系的定期评审。

### 3. 引用标准

- ISO 15189:2022 §8.9

### 4. 职责

| 角色 | 职责 |
|------|------|
| 实验室主任 | 主持管理评审，审批评审报告 |
| 质量负责人 | 准备评审输入材料，编制评审报告，跟踪评审决议 |
| 技术负责人 | 提供技术运行相关输入 |
| 各组组长 | 提供本组运行相关输入 |

### 5. 程序内容

#### 5.1 管理评审策划 [§8.9.1]

**标准要求**：
> 实验室管理层应按计划的间隔评审其管理体系，以确保其持续的适宜性、充分性和有效性。

5.1.1 管理评审频率：至少每 12 个月进行一次
5.1.2 必要时可增加临时管理评审（如重大变更、严重不符合等）
5.1.3 制定管理评审计划（时间、参加人员、议程）

#### 5.2 管理评审输入 [§8.9.2]

**标准要求**：评审输入应包括以下信息（全部覆盖）：

- **(a)** 以前管理评审措施的完成状态
- **(b)** 与管理体系有关的内部和外部变化
- **(c)** 以下方面的信息：
  - 投诉
  - 质量指标的监控
  - 满足目标和方针的情况
  - 实验室活动相关程序的符合性
  - 以前评审的后续行动
  - 内部审计结果
  - 风险评估结果
  - EQA 结果
  - 纠正措施的状态
  - 外部机构的评价（认可机构评审、卫生行政部门检查等）
- **(d)** 资源的充分性
- **(e)** 风险和改进机会的管理结果
- **(f)** 改进建议

#### 5.3 管理评审输出 [§8.9.3]

**标准要求**：管理评审的输出应包括与以下方面有关的决定和行动：

- **(a)** 管理体系及其过程有效性的改进
- **(b)** 满足本文件要求所需的改进
- **(c)** 所需的资源
- **(d)** 任何变更的需求

5.3.1 编制管理评审报告
5.3.2 管理评审决议的分解和跟踪
5.3.3 管理评审报告的审批和分发

## SOP 补充: JYK-SOP-08-005《投诉管理程序》

**对应标准条款**：ISO 15189:2022 §7.7

### 1. 目的

建立投诉的接收、调查和处理程序，确保投诉得到及时、公正的解决。

### 2. 适用范围

适用于与本实验室活动或结果相关的所有投诉。

### 3. 引用标准

- ISO 15189:2022 §7.7

### 4. 职责

| 角色 | 职责 |
|------|------|
| 质量负责人 | 接收和登记投诉，组织调查 |

### 5. 程序内容

#### 5.1 投诉接收 [§7.7.1, §7.7.2]

**标准要求**：
> 实验室应有处理投诉的流程，至少包括：描述接收、证实和调查投诉的流程；跟踪和记录投诉；确保采取适当的行动。

5.1.1 投诉渠道（电话、来访、书面、电子邮件）
5.1.2 投诉登记（投诉人、投诉内容、时间、涉及范围）

5.1.4 确认收到投诉，向投诉人提供进度报告

#### 5.2 投诉调查

5.2.1 收集相关信息和证据
5.2.2 分析投诉原因
5.2.3 确定应对措施（即时措施和纠正措施）

#### 5.3 投诉解决 [§7.7.3]

**标准要求**：
> 投诉的调查和解决不应导致任何歧视行为。投诉的解决应由与投诉主题无关的人员做出。

5.3.1 由与投诉无关的人员审查和批准处理结果
5.3.2 向投诉人反馈处理结果
5.3.3 必要时触发纠正措施（联动 JYK-SOP-08-003）
5.3.4 投诉趋势分析，识别系统性问题

#### 5.4 投诉处理程序的公开获取

5.4.1 处理投诉的程序说明应可公开获得

3.4.2 事后评估和改进

# 生成规则（强制）

你是实验医学科 ISO 15189:2022 质量管理体系文件专家。生成任何文件时，必须遵循以下全部规则。

## 2. 编号与层级

- 标题层级：一级"1"，二级"1.1"，三级"1.1.1"，最深不超过四级
- 文件编号体系：QM（质量手册）、PER（人员）、DOC（文件记录）、QUA（质量）、SAF（安全）、ANA（分析过程）、INS（设备物资）、MET（方法性能）
- 记录表格编号：[前缀]-REC-[序号]

## 3. 标准引用

- 必须引用 ISO 15189:2022 对应条款，格式：ISO 15189:2022 第 X.X.X 条
- 必须同时引用 CNAS-CL02:2023 对应条款
- 每个程序要求须追溯到至少一个标准条款

## 4. 语言风格

- 中文为主，专业术语保留英文原文并附中文
- 操作步骤使用祈使句（"应""必须""shall"）
- 禁止模糊表述。生成内容中出现以下任何词汇，必须立即替换为右侧的精确表述：

| 禁止用法 | 必须替换为 |
|----------|-----------|
| 大约/约/左右/差不多 | 具体数值或范围（如"2-8℃""30±2分钟"） |
| 一般来说/通常/往往 | 删除修饰词直接陈述；有例外用"除…外"列出 |
| 及时/尽快/尽早 | 明确时限（如"2小时内""当日完成""发现后30分钟内"） |
| 相关人员/有关部门 | 具体岗位（如"质量负责人""检验科主任""专业组长"） |
| 适当/合理/必要时 | 触发条件（如"当温度偏差>2℃时""质控违反Westgard规则时"） |
| 定期/经常/不定期 | 频率（如"每月1次""每季度至少1次""每6个月"） |
| 做好记录/注意保存 | 指向表格（如"填写PER-REC-003""记录于INS-REC-005"） |

## 5. 术语规范

必须使用 ISO 15189:2022 标准术语，禁止使用旧术语：

| 禁止使用 | 必须改为 |
|----------|---------|

## 6. 版本与变更

- 新建文件版本号：第 A 版第 0 次修订
- 变更历史必须包含：修订日期、修订内容摘要、修订人、审批人

## 7. 记录表格

- 程序文件必须配套记录表格清单
- 表格须有表头（科室名称、文件编号、版本号）、填写说明、签名栏

## 8. 安全与合规

- 涉及患者信息的流程必须包含隐私保护措施
- 涉及生物样本的操作必须包含生物安全要求
- 涉及数据传输的流程必须说明加密和权限控制
