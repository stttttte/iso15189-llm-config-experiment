# 设备校准记录表补充字段说明

## 一、补充字段概述

在现有设备校准记录表的基础上，需新增以下四个字段以满足 CNAS-CL01（等同采用 ISO/IEC 17025）及实验室质量管理体系的要求。这四个字段分别为：校准依据/方法、测量不确定度、下次校准日期、审核人/批准人。以下对各字段的定义、填写规范及业务逻辑进行详细说明。

## 二、字段详细设计

### 1. 校准依据/方法（calibration_method）

- **字段类型**：文本（VARCHAR 500）
- **是否必填**：是
- **说明**：记录本次校准所依据的国家标准、行业标准或校准规范编号及名称。例如"JJG 196-2006 常用玻璃量器检定规程"或"JJF 1001-2011 通用计量术语及定义"。当采用非标方法时，应注明方法来源及经过确认的文件编号。
- **填写要求**：至少包含标准编号和标准名称，多个依据之间用分号分隔。若为外部校准机构出具的证书，可填写证书中注明的校准方法。
- **界面建议**：提供常用校准标准的下拉候选列表，同时支持手动输入自定义内容。

### 2. 测量不确定度（measurement_uncertainty）

- **字段类型**：文本（VARCHAR 200）
- **是否必填**：是
- **说明**：记录校准结果的扩展不确定度，包含数值、单位及置信概率（通常为 k=2，约 95% 置信水平）。例如"U = 0.05 mL, k=2"或"U = 0.3 ℃, k=2"。该字段是实验室评估测量结果可靠性的核心参数。
- **填写要求**：必须注明不确定度数值、对应单位和包含因子 k 值。若校准证书中已给出不确定度，直接引用即可；若为内部校准，应按照 JJF 1059.1 的要求进行评定后填写。
- **界面建议**：提供格式提示"U = [数值] [单位], k=[包含因子]"，避免填写格式不统一。

### 3. 下次校准日期（next_calibration_date）

- **字段类型**：日期（DATE）
- **是否必填**：是
- **说明**：根据校准周期自动计算或手动指定的下一次校准到期日。该字段用于系统到期预警，确保设备在有效校准状态下使用。
- **填写要求**：日期格式为 YYYY-MM-DD。通常根据设备的校准周期（如 12 个月、6 个月）从本次校准日期自动推算。对于新设备首次校准或校准结果偏离较大的设备，可由技术负责人手动调整校准周期。
- **业务逻辑**：系统应在下次校准日期前 30 天自动发出提醒通知；超期未校准的设备应自动标记为"待校准"状态，并在检验模块中限制该设备的选用。

### 4. 审核人/批准人（reviewer_approver）

- **字段类型**：文本（VARCHAR 100），关联人员表
- **是否必填**：是
- **说明**：对校准记录进行审核和批准的责任人。审核人通常为质量监督员或计量管理员，负责核查校准过程的规范性和数据的准确性；批准人通常为技术负责人或授权签字人，负责确认校准结论的有效性。
- **填写要求**：从人员管理模块中选择具有相应授权资质的人员。审核人和批准人不得与校准执行人为同一人，以确保校准过程的独立性和公正性。
- **界面建议**：提供人员下拉选择框，仅显示具有"校准审核"或"校准批准"权限的人员列表。审核与批准可设为两个独立子字段，也可合并为一个字段以逗号分隔。

## 三、数据库变更

在现有 `calibration_record` 表中执行以下 ALTER 语句新增字段：

```sql
ALTER TABLE calibration_record ADD COLUMN calibration_method VARCHAR(500) NOT NULL DEFAULT '';
ALTER TABLE calibration_record ADD COLUMN measurement_uncertainty VARCHAR(200) NOT NULL DEFAULT '';
ALTER TABLE calibration_record ADD COLUMN next_calibration_date DATE;
ALTER TABLE calibration_record ADD COLUMN reviewer_approver VARCHAR(100) DEFAULT '';
```

以上四个字段的补充，将使设备校准记录表在溯源性、不确定度评估、周期管理和审批责任方面形成完整闭环，满足实验室认可准则对校准管理的全部关键要求。
